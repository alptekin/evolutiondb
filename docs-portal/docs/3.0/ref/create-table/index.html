CREATE TABLE

Define a new base table: its columns, types, per-column attributes, and table-level constraints: in the current schema, or populate one from a query result.

Synopsis

CREATE [ TEMPORARY | GLOBAL TEMPORARY ] TABLE [ IF NOT EXISTS ] [ schema . ]table_name (
    { column_name data_type [ column_attribute [ ... ] ]
    | table_constraint }
    [ , ... ]
)
[ table_option [ ... ] ]
[ INHERITS ( parent_table ) ]

-- Create a table from a query result (CTAS):
CREATE [ TEMPORARY | GLOBAL TEMPORARY ] TABLE [ IF NOT EXISTS ] [ schema . ]table_name
    [ ( column_name [ , ... ] ) ]
    [ IGNORE | REPLACE ] [ AS ] select_statement

-- Create a range partition child bound to an existing parent:
CREATE [ TEMPORARY | GLOBAL TEMPORARY ] TABLE [ IF NOT EXISTS ] table_name
    PARTITION OF parent FOR VALUES FROM ( lo ) TO ( hi )

where column_attribute is one of:
    NOT NULL | NULL
    DEFAULT value
    DEFAULT { gen_random_uuid() | gen_random_uuid_v7() | snowflake_id() | CURRENT_TIMESTAMP | ( expr ) }
    AUTO_INCREMENT [ ( start [ , increment ] ) ]
    IDENTITY [ ( start [ , increment ] ) ]
    PRIMARY KEY | KEY
    UNIQUE [ KEY ]
    CHECK ( expr )
    COMMENT 'string'
    CONSTRAINT name { UNIQUE | PRIMARY KEY | CHECK ( expr ) }
    GENERATED ALWAYS AS ( expr ) [ STORED | VIRTUAL ]
    AS ( expr ) [ STORED | VIRTUAL ]

and table_constraint is one of:
    [ CONSTRAINT name ] PRIMARY KEY ( column [ , ... ] )
    [ CONSTRAINT name ] UNIQUE ( column [ , ... ] )
    [ CONSTRAINT name ] CHECK ( expr )
    [ CONSTRAINT name ] FOREIGN KEY ( column [ , ... ] )
        REFERENCES [ schema . ]ref_table ( ref_column [ , ... ] ) [ fk_action [ ... ] ]
    KEY ( column [ , ... ] ) | INDEX ( column [ , ... ] )
    FULLTEXT { INDEX | KEY } ( column [ , ... ] )

and fk_action is one of:
    ON DELETE { CASCADE | SET NULL | RESTRICT | SET DEFAULT | NO ACTION }
    ON UPDATE { CASCADE | SET NULL | RESTRICT | SET DEFAULT | NO ACTION }
    MATCH { FULL | SIMPLE | PARTIAL }
    [ NOT ] DEFERRABLE [ INITIALLY { DEFERRED | IMMEDIATE } ]

and table_option is one of:
    AUTO_INCREMENT [ = ] n
    ON COMMIT { DELETE ROWS | PRESERVE ROWS }
    SHARD BY HASH ( column ) SHARDS n
    SHARD BY RANGE ( column ) ( shard_def [ , ... ] )
    PARTITION BY RANGE ( column )
    WITH SYSTEM VERSIONING
    WITH ( ttl_column = 'column_name' )

Description

CREATE TABLE creates a new, initially empty base table in the current schema of the current database. The table is owned by the issuing user. Each column is given a name, a data type, and an optional list of column attributes; table-level constraints and table options may follow the column list.

EvolutionDB resolves the current database and schema before creating the table. If the table name is qualified as schema.table_name, the leading name selects the target schema; otherwise the session's current schema (see SET SCHEMA) is used. If the target schema does not exist, the statement fails.

Columns and storage use EvolutionDB's binary tuple format: every record is prefixed with a sentinel byte and the table id, then a header, a null bitmap, and the column data. Type codes carry both a type family and a size (for example, VARCHAR(50) is encoded as 130050). A table may have at most 256 columns.

If a table with the same name already exists in the target schema, the statement fails with a "table already exists" error (SQLSTATE 42P07) unless IF NOT EXISTS is given, in which case the existing table is left untouched and the statement succeeds without error.

Create from a query (CTAS). When a select_statement follows the table name, the new table is populated from the query result. If an explicit column list is supplied, those names are used; otherwise column names and types are inferred from the SELECT output. IGNORE and REPLACE modify how rows that would violate uniqueness are handled during the bulk load.

Inheritance. INHERITS ( parent ) prepends the parent table's columns to the child's own columns. The parent must already exist, and a child column may not duplicate the name of an inherited column.

Partition child. The PARTITION OF parent FOR VALUES FROM (lo) TO (hi) form creates a range-partition child bound to an existing range-partitioned parent for the half-open integer range [lo, hi).

Parameters

TEMPORARY / GLOBAL TEMPORARY
Marks the table as temporary. TEMPORARY creates a session-local temporary table; GLOBAL TEMPORARY creates a global temporary table. Combine with the ON COMMIT table option to control row lifetime on transaction commit.
IF NOT EXISTS
Suppresses the "table already exists" error. If a table of the same name already exists in the target schema, no new table is created and no error is raised. (Note: the grammar spells this as the IF NOT EXISTS clause; it does not validate column compatibility with the existing table.)
schema.table_name / table_name
The name of the table to create, optionally schema-qualified. An unqualified name is created in the session's current schema.
column_name
The name of a column in the new table.
data_type
The column's data type. Supported types include the integer family (BIT, TINYINT, SMALLINT, MEDIUMINT, INT/INTEGER, BIGINT, each accepting an optional UNSIGNED/ZEROFILL modifier), approximate and exact numerics (REAL, DOUBLE, FLOAT, DECIMAL(p,s)), date/time types (DATE, TIME, TIMESTAMP, DATETIME, YEAR), character types (CHAR(n), VARCHAR(n), TINYTEXT, TEXT, MEDIUMTEXT, LONGTEXT), binary types (BINARY(n), VARBINARY(n), TINYBLOB, BLOB, MEDIUMBLOB, LONGBLOB), ENUM(...), SET(...), BOOLEAN, UUID, JSON, VECTOR(dim), and array types written as type[]. Character and text types accept an optional CHAR SET 'name' / COLLATE 'name' suffix.
NOT NULL / NULL
Declares whether the column may contain NULL. NOT NULL rejects NULLs; NULL (the default) allows them.
DEFAULT value
Supplies a default value used when an INSERT omits the column. The value may be a string, integer, float, or boolean literal. Function defaults gen_random_uuid(), gen_random_uuid_v7(), snowflake_id(), and the special value CURRENT_TIMESTAMP are recognized, as is a parenthesized expression default DEFAULT ( expr ).
AUTO_INCREMENT [ ( start [, increment ] ) ] / IDENTITY [ ( start [, increment ] ) ]
Auto-assigns increasing integer values for the column. With no arguments the sequence starts at 1 and increments by 1; an explicit start and optional increment may be given. IDENTITY is an alias for the same mechanism.
PRIMARY KEY / KEY (column attribute)
Marks the column as the table's primary key. KEY used as a column attribute is treated as PRIMARY KEY.
UNIQUE [ KEY ] (column attribute)
Requires the column's values to be unique across rows.
CHECK ( expr )
Defines a row-level check constraint that each row must satisfy. May be given inline on a column or as a table-level constraint.
COMMENT 'string'
Attaches a descriptive comment to the column.
GENERATED ALWAYS AS ( expr ) [ STORED | VIRTUAL ] / AS ( expr ) [ STORED | VIRTUAL ]
Defines a generated column whose value is computed from expr. STORED persists the computed value; VIRTUAL (the default when neither keyword is given) computes it on read.
CONSTRAINT name..
Assigns an explicit name to a UNIQUE, PRIMARY KEY, CHECK, or FOREIGN KEY constraint, whether written inline or at the table level. Unnamed CHECK constraints are auto-named table_check_n.
PRIMARY KEY ( column [.. ] ) (table constraint)
Declares one or more columns as the primary key. Supports composite keys.
UNIQUE ( column [.. ] ) (table constraint)
Declares a (possibly multi-column) uniqueness constraint.
FOREIGN KEY ( column [.. ] ) REFERENCES [ schema. ]ref_table ( ref_column [.. ] ) [ fk_action.. ]
Declares a referential integrity constraint linking local columns to columns in ref_table (optionally in another schema). Referential actions are configured with the fk_action clauses below.
fk_action, ON DELETE / ON UPDATE { CASCADE | SET NULL | RESTRICT | SET DEFAULT | NO ACTION }
Selects the action applied to referencing rows when the referenced row is deleted or updated.
fk_action, MATCH { FULL | SIMPLE | PARTIAL }
Sets the match semantics for composite foreign keys.
fk_action, [ NOT ] DEFERRABLE [ INITIALLY { DEFERRED | IMMEDIATE } ]
Controls whether constraint checking can be deferred to transaction commit, and the initial check timing.
KEY / INDEX ( column [.. ] )
Requests an index on the listed columns as part of the table definition. INDEX and KEY are accepted as synonyms in this position.
FULLTEXT { INDEX | KEY } ( column [.. ] )
Requests a full-text index on the listed columns.
AUTO_INCREMENT [ = ] n (table option)
Sets the starting value of the table's auto-increment sequence.
ON COMMIT { DELETE ROWS | PRESERVE ROWS } (table option)
For temporary tables, chooses whether rows are deleted or preserved at transaction commit.
SHARD BY HASH ( column ) SHARDS n / SHARD BY RANGE ( column ) (.. )
Distributes the table across n hash shards on column, or across explicit value ranges. Each range shard is written as SHARD name VALUES LESS THAN { 'value' | MAXVALUE } ON NODE id.
PARTITION BY RANGE ( column ) (table option)
Declares the table as range-partitioned on column. Child partitions are added with the PARTITION OF form.
WITH SYSTEM VERSIONING (table option)
Turns the table into a system-versioned (temporal) table. EvolutionDB auto-injects validity-period columns and links a table_history shadow table that retains superseded row versions.
WITH ( ttl_column = 'column_name' ) (table option)
Designates a TTL column. The auto-reclaim daemon prunes rows whose value in this column is older than the current time. The named column must exist in the table.
INHERITS ( parent_table )
Inherits the parent table's columns, which are prepended to the child's own columns. The parent must exist; duplicate column names are rejected.
PARTITION OF parent FOR VALUES FROM ( lo ) TO ( hi )
Creates a range-partition child of an existing partitioned parent, covering the half-open integer range from lo (inclusive) to hi (exclusive).
[ IGNORE | REPLACE ] [ AS ] select_statement
Populates the new table from the result of select_statement (CREATE TABLE AS). IGNORE skips rows that would conflict; REPLACE replaces them. The AS keyword is optional.

Examples

Create a simple table with a primary key, defaults, and a check:

CREATE TABLE users (
    id        INT PRIMARY KEY AUTO_INCREMENT,
    email     VARCHAR(255) NOT NULL UNIQUE,
    age       INT CHECK (age >= 0),
    status    VARCHAR(20) DEFAULT 'active',
    created   TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Idempotent creation and a UUID default:

CREATE TABLE IF NOT EXISTS sessions (
    id    UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    token TEXT NOT NULL
);

Composite primary key and a named foreign key with referential actions:

CREATE TABLE order_items (
    order_id  INT,
    product_id INT,
    qty       INT NOT NULL DEFAULT 1,
    PRIMARY KEY (order_id, product_id),
    CONSTRAINT fk_order FOREIGN KEY (order_id)
        REFERENCES orders (id) ON DELETE CASCADE
);

Generated column and a vector column:

CREATE TABLE docs (
    id       BIGINT IDENTITY,
    title    VARCHAR(200),
    title_len INT GENERATED ALWAYS AS (length(title)) STORED,
    embedding VECTOR(768)
);

Create from a query (CTAS), and a system-versioned table:

CREATE TABLE active_users AS
    SELECT id, email FROM users WHERE status = 'active';

CREATE TABLE accounts (
    id      INT PRIMARY KEY,
    balance DECIMAL(12,2)
) WITH SYSTEM VERSIONING;

Compatibility

EvolutionDB's CREATE TABLE blends SQL-standard, PostgreSQL, and MySQL conventions, and is not strictly conformant to any single one.

  • From the SQL standard / PostgreSQL: column definitions, PRIMARY KEY, UNIQUE, CHECK, FOREIGN KEY ... REFERENCES with ON DELETE/ON UPDATE actions and MATCH/DEFERRABLE clauses, GENERATED ALWAYS AS, INHERITS, IF NOT EXISTS, and CREATE TABLE AS are all supported.
  • From MySQL: AUTO_INCREMENT, KEY/INDEX and FULLTEXT declarations inside the column list, the COMMENT column attribute, UNSIGNED/ZEROFILL integer modifiers, the MySQL type spellings (TINYINT, MEDIUMTEXT, LONGBLOB, ENUM, SET, DATETIME, YEAR), the AS ( expr ) STORED|VIRTUAL generated-column shorthand, and the trailing table options syntax are MySQL-style.
  • EvolutionDB extensions: VECTOR(dim) columns, the gen_random_uuid_v7() and snowflake_id() default functions, WITH SYSTEM VERSIONING, WITH (ttl_column = ...), hash/range sharding (SHARD BY), and the integer-range PARTITION OF ... FOR VALUES child form are EvolutionDB-specific.
  • Differences and unsupported clauses: Unlike PostgreSQL, there is no UNLOGGED, no OF type, no EXCLUDE constraints, no per-column COLLATE using identifiers (a quoted string follows COLLATE instead), and no tablespace clause. The grammar accepts the NULL column attribute as an explicit no-op affirming nullability.
