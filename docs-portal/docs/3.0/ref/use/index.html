USE

USE — select the active database for the current session.

Synopsis

USE [ DATABASE ] database_name

Description

USE changes which database subsequent statements operate against. EvolutionDB resolves every table reference through the active database and schema pair, so issuing USE repoints all unqualified object lookups at the named database.

The target database must already exist in the system catalog. If it does not, the command fails with database "database_name" does not exist (SQLSTATE for undefined database) and the active database is left unchanged.

On success, switching the active database also resets the active schema to default. Any prior schema selection (for example one made with SET SCHEMA) is discarded by the switch; if you need a non-default schema in the new database you must select it again afterwards.

The optional DATABASE keyword is purely cosmetic — USE mydb and USE DATABASE mydb are accepted by the grammar and behave identically.

Note: USE changes session state, not stored schema. It is not transactional in the sense of being rolled back by a failed statement; it takes effect immediately for the connection that issues it and persists until the next USE (or until the connection closes).

Parameters

DATABASE
Optional noise keyword. Present purely for readability and MySQL-style familiarity; it does not alter behavior. Both USE name and USE DATABASE name parse to the same action.
database_name
The name of the database to make active. It must be an existing database registered in the system catalog. Lookups are performed against the catalog, and a missing database produces an error rather than creating one. After a successful switch, the active schema is reset to default.

Examples

Switch to a database named sales using the short form:

USE sales;

The same switch written with the optional DATABASE keyword:

USE DATABASE sales;

Create a database and then begin working inside it. Because the switch resets the schema to default, the CREATE TABLE below lands in sales.default:

CREATE DATABASE sales;
USE sales;
CREATE TABLE orders (
    id      INT PRIMARY KEY,
    total   DECIMAL(12, 2)
);

Select a non-default schema after switching databases. USE first resets the schema, so the explicit SET SCHEMA must come after it:

USE DATABASE sales;   -- active schema is now "default"
SET SCHEMA archive;  -- active schema is now "archive"

Compatibility

USE is not part of the SQL standard. EvolutionDB's form follows the MySQL convention of a session-level command for selecting the current database, and additionally accepts the optional DATABASE noise keyword.

PostgreSQL has no USE statement. In PostgreSQL a connection is bound to one database for its lifetime, and switching databases requires opening a new connection (for example the psql meta-command \c). EvolutionDB instead lets a single connection move between databases at runtime.

A further difference from PostgreSQL: EvolutionDB's default schema is named default, not public, and a successful USE always resets the active schema to default.
