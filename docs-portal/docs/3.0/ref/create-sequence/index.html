CREATE SEQUENCE

Define a new sequence generator: a named, persistent counter that produces a monotonic series of BIGINT values via the nextval(), currval(), setval() and lastval() functions.

Synopsis

CREATE SEQUENCE [ IF NOT EXISTS ] name
    [ INCREMENT [ BY ] increment ]
    [ MINVALUE minvalue ]
    [ MAXVALUE maxvalue ]
    [ START [ WITH ] start ]
    [ CYCLE | NOT CYCLE ]

Description

CREATE SEQUENCE creates a new sequence-number generator. The sequence is stored as a persistent object in the system catalog of the current schema, and survives server restarts.

After creation, the sequence is manipulated through the sequence functions. nextval(name) advances the sequence and returns its new value; the very first call to nextval() on a freshly created sequence returns the START value (the sequence is not yet "called"). Subsequent calls add increment to the current value. currval() returns the value most recently obtained, lastval() returns the value of the most recently used sequence in the session, and setval() resets the current value.

If the supplied name already names a sequence in the current schema, an error is raised unless IF NOT EXISTS is given, in which case a notice is emitted and no new sequence is created. Note that IF NOT EXISTS only checks whether a sequence of the same name exists; it does not verify that the existing definition matches the requested one.

Each call to nextval() is serialized by an internal mutex and immediately persisted to the catalog, so concurrent sessions never receive the same value. Because the new value is written before it is returned, sequence values consumed inside a transaction are not rolled back if that transaction aborts; gaps in the series are therefore normal and expected.

When increment is positive the sequence is ascending; when it is negative the sequence is descending. On reaching maxvalue (ascending) or minvalue (descending), a sequence declared CYCLE wraps around to minvalue or maxvalue respectively, while a non-cycling sequence raises an error.

Parameters

IF NOT EXISTS
Do not raise an error if a sequence with the same name already exists in the current schema. A notice is issued instead. The pre-existing sequence is left unchanged.
name
The name of the sequence to create. The sequence is created in the current schema. Names are not schema-qualified in this form.
INCREMENT [ BY ] increment
The value added to the current sequence value on each call to nextval(). A positive value makes an ascending sequence, a negative value a descending one. The BY keyword is optional. Defaults to 1.
MINVALUE minvalue
The minimum value the sequence can generate. Defaults to 1. For a descending sequence this is the value used as the cycle wrap target when CYCLE is set, and the floor at which a non-cycling descending sequence errors out.
MAXVALUE maxvalue
The maximum value the sequence can generate. Defaults to 2147483647 (231−1). For an ascending sequence this is the cycle wrap target when CYCLE is set, and the ceiling at which a non-cycling ascending sequence errors out.
START [ WITH ] start
The first value returned by the sequence, the value yielded by the first nextval() call. The WITH keyword is optional. If omitted, the start value defaults to minvalue.
CYCLE / NOT CYCLE
CYCLE allows the sequence to wrap around when it reaches maxvalue (ascending) or minvalue (descending). NOT CYCLE (the default) causes nextval() to raise an error once a bound is reached.

Note. The clause order shown in the Synopsis is conventional, not mandatory: the grammar accepts the options in any order, and a later occurrence of the same option overrides an earlier one. Sequence values are 64-bit signed integers (BIGINT), but the default MAXVALUE is 2147483647; raise MAXVALUE explicitly if you need the full 64-bit range.

Examples

Create an ascending sequence starting at 1, incrementing by 1 (all defaults):

CREATE SEQUENCE order_id_seq;

Draw values from it:

SELECT nextval('order_id_seq');   -- 1  (the START value)
SELECT nextval('order_id_seq');   -- 2
SELECT currval('order_id_seq');   -- 2

Create a sequence that starts at 1000 and counts up by 10:

CREATE SEQUENCE invoice_seq
    START WITH 1000
    INCREMENT BY 10;

Create a bounded, cycling sequence (1 → 5, then back to 1):

CREATE SEQUENCE day_slot_seq
    MINVALUE 1
    MAXVALUE 5
    CYCLE;

Create a descending sequence:

CREATE SEQUENCE countdown_seq
    START 100
    INCREMENT -1
    MINVALUE 0;

Create idempotently, skip if it already exists:

CREATE SEQUENCE IF NOT EXISTS session_seq;

Reposition a sequence with setval() so the next nextval() returns 200:

SELECT setval('invoice_seq', 200);

Compatibility

CREATE SEQUENCE conforms to the spirit of the SQL standard, with the following EvolutionDB-specific differences:

  • EvolutionDB supports the clauses INCREMENT [BY], MINVALUE, MAXVALUE, START [WITH] and CYCLE / NOT CYCLE, plus the EvolutionDB extension IF NOT EXISTS.
  • The standard's bare keyword NO CYCLE is written NOT CYCLE in EvolutionDB. The forms NO MINVALUE and NO MAXVALUE are not supported; omit the clause to accept the default instead.
  • The PostgreSQL clauses AS data_type, CACHE, OWNED BY and TEMPORARY/UNLOGGED are not accepted by the grammar. Sequences are always persistent BIGINT generators, and every nextval() writes through to the catalog (no client-side cache).
  • The standard reserves nextval/currval as expressions; like PostgreSQL, EvolutionDB exposes them as functions taking the sequence name as a string argument, e.g. nextval('seq').
  • The default MAXVALUE is 2147483647 rather than the 64-bit maximum; raise it explicitly when a larger ceiling is required.
