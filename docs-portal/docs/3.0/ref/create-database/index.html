CREATE DATABASE

Create a new, empty database and register it in the system catalog, automatically provisioning a schema named default inside it.

Synopsis

CREATE DATABASE [ IF NOT EXISTS ] database_name

Description

CREATE DATABASE creates a new database. A database is the top-level container in EvolutionDB's catalog hierarchy: it holds schemas, which in turn hold tables and other objects. The statement records the new database in the system catalog and assigns it a catalog-generated database id.

As part of creation, EvolutionDB immediately provisions a schema named default inside the new database, so the database is ready to receive objects without a separate CREATE SCHEMA step.

Creating a database does not switch the session to it. The session's current database is unchanged; use USE to make the new database current before creating objects in it.

If a database with the same name already exists, the statement fails with a "database already exists" error (SQLSTATE 42E04) unless IF NOT EXISTS is given, in which case the existing database is left untouched and the statement succeeds without raising an error. If the catalog cannot allocate the database for any other reason, the statement fails with an I/O error (SQLSTATE 58030).

On a freshly initialized instance the catalog is bootstrapped with a default database named evosql (and a default schema within it), so at least one database always exists before any CREATE DATABASE is issued.

Parameters

IF NOT EXISTS
Suppresses the "database already exists" error. If a database with the given name is already present in the catalog, no new database is created and the statement completes successfully without modifying the existing one. Without this clause, attempting to create a duplicate database is an error.
database_name
The name of the database to create. It must be unique among existing databases; name matching against the catalog is case-sensitive for the lookup performed on creation.

Examples

Create a database:

CREATE DATABASE shop;

Create a database only if it does not already exist (idempotent):

CREATE DATABASE IF NOT EXISTS analytics;

Create a database, then switch to it and start defining objects in its default schema:

CREATE DATABASE reporting;
USE reporting;
CREATE TABLE events (
    id   BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(120) NOT NULL
);

Compatibility

The SQL standard does not define CREATE DATABASE; it is a PostgreSQL/MySQL extension. EvolutionDB implements a deliberately minimal form of the command.

  • Versus PostgreSQL: PostgreSQL's CREATE DATABASE accepts a long list of options such as OWNER, TEMPLATE, ENCODING, LOCALE, LC_COLLATE, LC_CTYPE, TABLESPACE, CONNECTION LIMIT, and IS_TEMPLATE. EvolutionDB supports none of these, they are not accepted by the grammar. The only optional element is the IF NOT EXISTS guard.
  • Versus MySQL: MySQL allows CREATE {DATABASE | SCHEMA} with CHARACTER SET and COLLATE options; EvolutionDB does not accept those options on CREATE DATABASE. In EvolutionDB, CREATE DATABASE and CREATE SCHEMA are distinct commands at different levels of the hierarchy (a schema lives inside the current database), not synonyms.
  • EvolutionDB behavior: Creating a database also creates a schema named default within it. The new database does not become the session's current database; an explicit USE is required.
