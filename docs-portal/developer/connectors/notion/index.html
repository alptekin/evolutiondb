Notion connector

Pull every Notion page shared with an integration token into EvolutionDB long-term memory, one page per memory record.

What it does

evolutiondb-notion-sync is a one-way connector that walks every page a Notion integration can see, flattens each page's body text, and writes one record per page into an EvolutionDB memory store. It is the same record-into-memory pattern used by the other *-sync connectors, and the records it writes are the kind the MCP server reads back as agent long-term memory.

The flow for each sync pass:

  • Call the Notion /search endpoint to list pages, newest first.
  • Fetch each page's top-level child blocks via /blocks/{id}/children and join their plain text.
  • Build a JSON record (fact, title, text, page_id, url, timestamps, tags) and store it under a per-page key.
  • Track a watermark (the newest last_edited_time seen) so the next pass stops as soon as it reaches already-synced pages, making repeated runs incremental.
info
The connector only fetches top-level child blocks of each page. Nested children (toggles, sub-lists, columns) are not recursively expanded, and non-text blocks (images, embeds, tables) contribute no body text.

Install

Distribution name on PyPI metadata is evolutiondb-notion-sync; the import package is notion_sync. Requires Python 3.9+.

# installs the console script `evolutiondb-notion-sync`
pip install evolutiondb-notion-sync

Runtime dependencies (declared in pyproject.toml):

  • psycopg[binary]>=3.1, connects to EvolutionDB over the PostgreSQL wire protocol.
  • evolutiondb-pii>=0.1.1, optional PII masking; the connector degrades gracefully if it cannot be imported (see Notes).

The Notion HTTP client itself uses only the Python standard library (urllib), no third-party Notion SDK.

Configure (env vars / auth)

Configuration is read entirely from environment variables by the Config class. A .env file (path set by --env-file, default .env in the working directory) is loaded first with os.environ.setdefault, so real environment variables always win over .env entries.

VariableDefaultPurpose
NOTION_TOKEN(required)Notion internal-integration secret. Empty value raises an error.
EVOSQL_HOST127.0.0.1EvolutionDB host.
EVOSQL_PORT5433EvolutionDB PG-wire port.
EVOSQL_USERadminDatabase user.
EVOSQL_PASSWORDadminDatabase password.
EVOSQL_DATABASEevosqlDatabase name.
MCP_USER_IDdefault_userMemory namespace records are written under.
NOTION_MEMORY_STOREmcp_memTarget memory store name.

To set up Notion access (from the package docstring):

  1. Create an internal integration at https://www.notion.so/my-integrations and copy its secret.
  2. Export it as NOTION_TOKEN in .env or the calling shell.
  3. On each Notion page or database you want indexed, open the share menu and add the integration. The sync only sees pages explicitly shared with it.
# .env
NOTION_TOKEN=secret_xxxxxxxxxxxxxxxxxxxx
EVOSQL_HOST=127.0.0.1
EVOSQL_PORT=5433
EVOSQL_DATABASE=evosql
MCP_USER_ID=default_user
NOTION_MEMORY_STORE=mcp_mem

Usage

The console script is the primary entry point. It supports four flags, parsed in main():

FlagEffect
--onceRun a single sync pass and exit.
--interval NLoop forever, sleeping N seconds between passes (floored to a minimum of 60).
--dry-runWalk Notion and count pages without opening a DB connection or writing any records.
--env-file PATH.env path to load (default .env).

If neither --once nor --interval is given, a single pass runs (the same as --once).

# one pass, write to memory
evolutiondb-notion-sync --once

# walk Notion but write nothing - useful to verify the token works
evolutiondb-notion-sync --dry-run

# continuous daemon: a pass every 5 minutes
evolutiondb-notion-sync --interval 300

# load a non-default env file
evolutiondb-notion-sync --once --env-file /etc/evosql/notion.env

Each pass prints a single JSON line to stdout. On success it reports the counters (pages, skipped, errors); on failure it reports the error and the process returns exit code 4. 

# success
{"ok": true, "pages": 12, "skipped": 0, "errors": 0}

# failure (e.g. NOTION_TOKEN unset) - exit code 4
{"ok": false, "error": "NOTION_TOKEN is not set. ..."}

SIGINT and SIGTERM are handled: in interval mode they set a stop flag so the current sleep is interrupted and the loop exits cleanly after the in-flight pass.

As a library

You can drive a pass directly without the CLI:

from notion_sync.sync import Config, sync_once

cfg = Config()                  # reads env vars
counts = sync_once(cfg, dry_run=False)
print(counts)                    # {"pages": .., "skipped": .., "errors": ..}

Reference

sync module

class Config()

Reads all settings from environment variables on construction. Attributes: token, evosql_host, evosql_port, evosql_user, evosql_pass, evosql_db, user_id, store.

sync_once(cfg: Config, *, dry_run: bool = False) -> Dict[str, int]

Runs a single sync pass. Raises RuntimeError if cfg.token is empty. Iterates NotionClient.search_pages(), stops once it reaches the stored watermark, fetches each page's blocks, builds a record, optionally PII-protects it, and writes it to the memory store. Advances the watermark to the newest last_edited_time seen. Returns counters {"pages", "skipped", "errors"}. With dry_run=True no DB connection is opened and nothing is written.

main(argv: Optional[List[str]] = None) -> int

CLI entry point (console script evolutiondb-notion-sync). Parses flags, loads the env file, installs signal handlers, and runs one pass or an interval loop. Returns the process exit code.

Internal helpers in the module: _load_dotenv(path) (minimal .env reader), _build_record(page, body_text) (assembles the stored record; returns None if the page has no last_edited_time), _page_key(page_id) (record key notion_page_<id-without-dashes>), _short(s, n=240) (truncates the fact preview), and _protect(rec, main_key) (PII masking, see Notes).

api module

class NotionClient(token: str, user_agent: str = "evolutiondb-notion-sync")

Tiny Notion REST client built on urllib. Talks to https://api.notion.com/v1 with Notion-Version: 2022-06-28. Retries once on HTTP 429, honoring the Retry-After header.

NotionClient.search_pages() -> Iterator[Dict]

Yields every page the integration can see, newest first, by paging through POST /search (filtered to object = page, sorted descending by last_edited_time, page size 100) and following next_cursor.

NotionClient.page_blocks(page_id: str, page_size: int = 100) -> List[Dict]

Returns the top-level child blocks of a page via GET /blocks/{page_id}/children, following pagination cursors.

page_title(page: Dict) -> str

Extracts a human title from a page object by finding the property of type title. Returns "(untitled)" when none is present.

block_plain(block: Dict) -> str

Returns the plain text of a single block for the supported rich-text block types (paragraph, heading_1/2/3, bulleted_list_item, numbered_list_item, to_do, quote, callout, toggle, and code). Any other block type yields an empty string.

rich_text_plain(items: Optional[List[Dict]]) -> str

Concatenates the plain_text of a Notion rich-text array.

NotionError(status, msg, url) is raised for non-retryable HTTP errors and carries the HTTP status.

state module

class MemoryStore(host, port, user, password, database, store, namespace)

EvolutionDB memory backend (a mirror of the sibling connectors' backend). Connects via psycopg in autocommit mode and best-effort runs CREATE MEMORY STORE on init. All operations go through a small reconnect-with-backoff retry (3 attempts) that re-establishes the connection on OperationalError/InterfaceError.

MemoryStore.get_watermark_iso() -> Optional[str]

Reads the stored last_edited_time watermark for this namespace (key notion_state_latest).

MemoryStore.set_watermark_iso(iso: str) -> None

Persists the watermark using MEMORY PUT INTO <store>.

MemoryStore.put_record(key: str, record: Dict) -> None

Serializes record to JSON and writes it under (namespace, key) with MEMORY PUT INTO <store>.

Notes & limitations

  • One-way, page-level only. The connector reads from Notion and writes to EvolutionDB; it never writes back to Notion. It indexes pages (objects of type page): databases are not enumerated as records, and only top-level page blocks are flattened into the body text.
  • Incremental via watermark. Because /search returns newest-first and the loop breaks at the stored watermark, an interrupted or repeated run resumes from the last synced page rather than re-walking everything. The first run against a populated workspace walks all visible pages.
  • PII masking is optional. If evolutiondb-pii is installed, the fields fact, title, and text are passed through protect_record before storage and any companion records are written too. If the import fails, records are stored unmasked: there is no warning, the masking step is simply skipped.
  • SQL value escaping. state.py builds MEMORY PUT/SELECT statements by string interpolation with a local _e() helper that strips control characters and doubles single quotes. Namespace/store/key values flow through it; treat NOTION_MEMORY_STORE and MCP_USER_ID as trusted configuration, not untrusted input.
  • Interval floor. --interval values below 60 are clamped to 60 seconds.
  • Version note. pyproject.toml declares version 0.1.1 while notion_sync/__init__.py still reports __version__ = "0.1.0".
warning
The integration only sees pages explicitly shared with it. If a sync pass reports "pages": 0 and your token is valid, the most common cause is that no pages have been shared with the integration in Notion's share menu.

See also the MCP server that consumes these memory records, and the broader EvolutionDB wiki.
