Slack connector

A read-only *-sync connector that pulls the Slack DMs, group DMs, and channels you belong to into EvolutionDB long-term memory, so an AI assistant can recall them later through search_memory.

What it does

The package evolutiondb-slack-sync walks every conversation the token's user is part of: public channels they've joined, private channels, DMs, and group DMs, and persists each message into an EvolutionDB MCP memory store. The MCP server then surfaces those messages through search_memory, so an assistant can answer questions like "what did Ahmet say about the database migration last week?" weeks later.

Concretely, each pass:

  • Lists every conversation via conversations.list (types public_channel,private_channel,mpim,im, archived excluded).
  • For each conversation, fetches conversations.history newer than that channel's stored watermark (a Slack ts).
  • For any parent message that opens a thread (thread_ts == ts and reply_count > 0), walks conversations.replies to capture the thread.
  • Writes one record per message, keyed slack_msg_<channel>_<ts>. The key is deterministic, so re-sync UPSERTs rather than duplicating.
  • Advances a per-channel watermark so the next pass only reads new messages.

Records carry the channel display name, sender display name, channel type, and thread_ts when the message lives inside a thread, so downstream search_memory queries can filter on them.

What is not indexed

The sync loop deliberately skips:

  • System messages with subtype channel_join, channel_leave, bot_add, or bot_remove, pure noise.
  • Messages that have neither text nor attachments (counted as skipped). Per the README, file/image binaries are not stored, only the text caption is captured, and reactions are not indexed.

Install

The distribution name on PyPI is evolutiondb-slack-sync; the importable package is slack_sync. It requires Python 3.9+ and depends only on psycopg[binary]>=3.1 (the Slack client itself uses the standard library urllib, no slack_sdk dependency).

# From PyPI
pip install evolutiondb-slack-sync

# Or editable, from this repo
pip install -e client/slack-sync

Installing exposes a single console script, evolutiondb-slack-sync (entry point slack_sync.__main__:main).

Configure (env vars / auth)

Configuration is entirely through environment variables. The CLI loads a dotenv file before reading them, by default .env in the working directory, overridable with --env-file. Existing process environment values win (the loader uses setdefault, so it never overwrites what is already set).

Slack authentication

Auth is a single User OAuth token (xoxp-…), created at https://api.slack.com/apps → Create New App → From scratch. Add the following read-only user-token scopes under OAuth & Permissions, install the app, and copy the User OAuth Token into SLACK_USER_TOKEN:

channels:history    channels:read
groups:history      groups:read
im:history          im:read
mpim:history        mpim:read
users:read

These are user scopes, not bot scopes: the package walks the conversations the human is actually part of, so no workspace-admin approval is needed. The README notes that sending is opt-in and off by default; staying with the scopes above keeps the token strictly read-only. (The outbox action loop's reply path additionally requires the chat:write scope plus EVOSQL_SEND_ENABLED=1 / EVOSQL_SEND_CHANNELS=slack, per ADR-004, but that send path lives outside this package.)

Usage

The CLI has five flags: --once, --interval <seconds>, --since <duration> (default 3d), --dry-run, and --env-file <path> (default .env).

# First run - pull the last three days, then exit.
evolutiondb-slack-sync --once --since 3d

# Daemon mode: pass, then sleep --interval seconds, repeat.
evolutiondb-slack-sync --interval 600

# Auth/scope check only - no DB writes.
evolutiondb-slack-sync --once --dry-run

If --once is given, or no --interval is supplied, exactly one pass runs. In daemon mode the interval is clamped to a minimum of 60 seconds, and SIGTERM/SIGINT request a graceful stop after the current pass finishes.

Each pass prints a one-line JSON summary to stdout. On success:

{"ok": true, "channels": 28, "messages": 412, "replies": 96, "skipped": 14, "errors": 0}

On failure the line is {"ok": false, "error": "<message>"}. (The figures above are the README's illustrative example, not guaranteed counts.)

--since syntax

--since takes a duration of the form <n><unit>, where unit is one of s, m, h, d, w (e.g. 30s, 5m, 1h, 3d, 2w). It is converted to a Unix-seconds floor that becomes Slack's oldest bound. A garbage value exits with code 2. Note: --since is only the default floor: for channels that already have a stored watermark, the watermark takes precedence.

Record shape

Each message is stored as a JSON record under the key slack_msg_<channel>_<ts>. Fields written by _build_record:

{
  "fact":         "Alice in #general: hi all",
  "source":       "slack",
  "kind":         "message",        // or "thread_reply"
  "channel_id":   "C123",
  "channel_name": "#general",
  "channel_type": "public_channel",  // im | mpim | private_channel | public_channel
  "ts":           "1715594000.000200",
  "thread_ts":    null,
  "sender_id":    "U1",
  "sender":       "Alice",
  "text":         "hi all",
  "created_at":   "2024-05-13T08:33:20Z",
  "tags":         ["slack", "#general"]
}

Message text is clipped to ~4000 characters (overflow is truncated and suffixed with …[clipped]). Thread replies set kind to thread_reply, carry the parent's thread_ts, and add a "thread" tag.

Reference

The package is three small modules: slack_sync.sync (config, record building, sync loop, CLI), slack_sync.api (Slack Web API client), and slack_sync.state (EvolutionDB memory backend).

slack_sync.sync

def main(argv: Optional[list] = None) -> int

CLI entry point (the evolutiondb-slack-sync console script). Parses arguments, loads the dotenv file, builds Config, then runs one pass or loops on --interval. Returns a process exit code: 2 for a bad --since or missing token, 4 if a pass raises, otherwise 0.

def sync_once(cfg: Config, *, since_default: Optional[float], dry_run: bool = False) -> Dict[str, int]

Runs a single sync pass. Constructs a SlackClient and (unless dry_run) a MemoryStore, iterates every conversation, persists messages and thread replies, advances watermarks, and returns the counter dict (channels, messages, replies, skipped, errors). Per-channel SlackErrors are logged to stderr and counted as errors without aborting the pass. Raises RuntimeError if the token is empty.

def parse_since(text: str) -> float

Parses a <n><s|m|h|d|w> duration into a Unix-seconds float relative to now (UTC). Raises ValueError on anything else.

class Config

Reads all environment variables into typed attributes (token, evosql_host, evosql_port, evosql_user, evosql_pass, evosql_db, user_id, store, poll_secs). Constructed after the dotenv file is loaded.

Internal helpers: _build_record(...) assembles the record dict shown above; _ts_to_iso(ts) converts a Slack ts to an ISO-8601 Z string (or None); _short(s, n=4000) clips long text; _load_dotenv(path) populates the environment without overwriting existing values.

slack_sync.api

class SlackClient(token: str, user_agent: str = "evolutiondb-slack-sync")

A thin Slack Web API client built on urllib only. Authenticates every request with Authorization: Bearer <token>. On an HTTP 429 it honours the Retry-After header and retries exactly once; any other failed response raises SlackError.

SlackClient.list_conversations() -> Iterator[Dict]

Cursor-paginates conversations.list across all four conversation types (archived excluded), yielding one conversation dict at a time.

SlackClient.history(channel: str, oldest: Optional[float] = None) -> Iterator[Dict]

Pages conversations.history for one channel, buffers the batch, and yields messages sorted chronologically by ts so watermark logic stays monotonic.

SlackClient.replies(channel: str, thread_ts: str, oldest: Optional[float] = None) -> Iterator[Dict]

Pages conversations.replies for one thread, skipping the parent message (already captured by history), and yields the replies in chronological order.

SlackClient.user_name(user_id: str) -> str

Resolves a U… id to a display name via users.info, with an in-memory cache. Falls back to display_name → real_name → name → the raw id, and returns the id itself if the lookup fails (deleted user / missing scope).

def conversation_display_name(conv: Dict, client: SlackClient) -> str

Produces a human label for a conversation row: a DM resolves to the other user's display name, a group DM to its purpose.value, a channel to #name.

class SlackError(code: str, msg: str, url: str)

Raised for non-ok Slack responses and HTTP errors; exposes .code and .url. Common .code values seen in practice include not_in_channel, channel_not_found, and missing_scope.

slack_sync.state

class MemoryStore(host, port, user, password, database, store, namespace)

The EvolutionDB memory backend. Connects via psycopg (autocommit) and best-effort issues CREATE MEMORY STORE <store> on init. All operations run through a small reconnect-with-backoff wrapper (3 attempts) that recovers from dropped connections. Each connector keeps its own MemoryStore so packages stay dependency-free of one another.

MemoryStore.get_watermark(channel: str) -> Optional[str]

Reads the stored Slack ts watermark for a channel (key slack_state_<channel>), or None if not yet seen.

MemoryStore.set_watermark(channel: str, ts: str) -> None

Writes the channel watermark with MEMORY PUT INTO <store>, storing the Slack ts verbatim so microsecond precision round-trips.

MemoryStore.put_record(key: str, record: Dict) -> None

Serializes a record dict to JSON and writes it with MEMORY PUT INTO <store> under the configured namespace.

Notes and limitations

  • Beta. The package is version 0.1.0 and marked Development Status 4 - Beta. There is no slack_sdk dependency, the API client is hand-rolled on urllib.
  • Read-only by design. The scopes above grant no write access. There is no posting/sending code in this package; the opt-in reply path lives in the outbox action loop (see ADR-004) and additionally needs chat:write.
  • Rate limits. Slack throttles conversations.history at Tier 3 (50+ req/min per workspace). The client retries a 429 exactly once after sleeping for Retry-After. For very active workspaces, raise SLACK_POLL_INTERVAL rather than looping tighter.
  • SQL building uses string interpolation. state.py escapes values with a local _e() helper (newline/tab stripping + single-quote doubling) and interpolates them into the MEMORY PUT / SELECT statements rather than using parameter binding.
  • Privacy. The user-scope token reads exactly the conversations the human can already see, no privilege escalation, but once a message lands in EvolutionDB, anyone with the matching MCP_USER_ID namespace can recall it via search_memory. Confirm that piping work Slack into a personal memory store is acceptable before enabling it.
  • Tests. tests/test_sync.py covers the pure functions only (parse_since, _ts_to_iso, _build_record); there are no integration tests against a live Slack workspace or database in the package.
