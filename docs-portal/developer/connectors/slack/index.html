Slack connector

A read-only *-sync connector. It pulls the Slack DMs, group DMs, and channels you belong to into EvolutionDB long-term memory. An AI assistant can then recall them later through search_memory.

What it does

The package evolutiondb-slack-sync walks every conversation the token's user is part of. This includes public channels they have joined, private channels, DMs, and group DMs. It saves each message into an EvolutionDB MCP memory store. The MCP server then shows those messages through search_memory. An assistant can answer questions like "what did Ahmet say about the database migration last week?" weeks later.

Each pass does this work.

  • Lists every conversation via conversations.list. It uses the types public_channel,private_channel,mpim,im and leaves out archived ones.
  • For each conversation, fetches conversations.history newer than that channel's stored watermark. The watermark is a Slack ts.
  • Walks conversations.replies for any parent message that opens a thread. A thread parent has thread_ts == ts and reply_count > 0.
  • Writes one record per message, keyed slack_msg_<channel>_<ts>. The key is always the same for a message. So a re-sync UPSERTs instead of making duplicates.
  • Advances a per-channel watermark so the next pass only reads new messages.

Records carry the channel display name, sender display name, channel type, and thread_ts when the message is inside a thread. Later search_memory queries can filter on these fields.

What is not indexed

The sync loop skips these on purpose.

  • System messages with subtype channel_join, channel_leave, bot_add, or bot_remove. These are just noise.
  • Messages that have no text and no attachments. These count as skipped. The connector does not store file or image binaries. It only saves the text caption. Reactions are not indexed.

Install

The distribution name on PyPI is evolutiondb-slack-sync. The importable package is slack_sync. It needs Python 3.9+ and depends only on psycopg[binary]>=3.1. The Slack client uses the standard library urllib, so there is no slack_sdk dependency.

# From PyPI
pip install evolutiondb-slack-sync

# Or editable, from this repo
pip install -e client/slack-sync

Installing gives you one console script named evolutiondb-slack-sync. Its entry point is slack_sync.__main__:main.

Configure (env vars / auth)

You configure everything through environment variables. The CLI loads a dotenv file before it reads them. By default this is .env in the working directory. You can change it with --env-file. Values already in the process environment win. The loader uses setdefault, so it never overwrites what is already set.

Slack authentication

Auth uses one User OAuth token in the form xoxp-…. You create it at https://api.slack.com/apps. Select Create New App, then From scratch. Add the read-only user-token scopes below under OAuth & Permissions. Then install the app and copy the User OAuth Token into SLACK_USER_TOKEN.

channels:history    channels:read
groups:history      groups:read
im:history          im:read
mpim:history        mpim:read
users:read

These are user scopes, not bot scopes. The package walks the conversations the human is part of. So you do not need workspace-admin approval. Sending is opt-in and off by default. If you keep only the scopes above, the token stays read-only. The outbox action loop's reply path also needs the chat:write scope plus EVOSQL_SEND_ENABLED=1 and EVOSQL_SEND_CHANNELS=slack, per ADR-004. That send path lives outside this package.

Usage

The CLI has five flags. They are --once, --interval <seconds>, --since <duration> with default 3d, --dry-run, and --env-file <path> with default .env.

# First run - pull the last three days, then exit.
evolutiondb-slack-sync --once --since 3d

# Daemon mode: pass, then sleep --interval seconds, repeat.
evolutiondb-slack-sync --interval 600

# Auth/scope check only - no DB writes.
evolutiondb-slack-sync --once --dry-run

Exactly one pass runs if you give --once or do not supply --interval. In daemon mode the interval has a minimum of 60 seconds. SIGTERM and SIGINT ask for a clean stop after the current pass finishes.

Each pass prints a one-line JSON summary to stdout. A success looks like this.

{"ok": true, "channels": 28, "messages": 412, "replies": 96, "skipped": 14, "errors": 0}

On failure the line is {"ok": false, "error": "<message>"}. The numbers above are just an example from the README. They are not guaranteed counts.

--since syntax

--since takes a duration in the form <n><unit>. The unit is one of s, m, h, d, w. Some examples are 30s, 5m, 1h, 3d, and 2w. The connector turns it into a Unix-seconds floor and passes that as Slack's oldest bound. A bad value exits with code 2. --since is only the default floor. For channels that already have a stored watermark, the watermark wins.

Record shape

Each message is stored as a JSON record under the key slack_msg_<channel>_<ts>. These are the fields written by _build_record.

{
  "fact":         "Alice in #general: hi all",
  "source":       "slack",
  "kind":         "message",        // or "thread_reply"
  "channel_id":   "C123",
  "channel_name": "#general",
  "channel_type": "public_channel",  // im | mpim | private_channel | public_channel
  "ts":           "1715594000.000200",
  "thread_ts":    null,
  "sender_id":    "U1",
  "sender":       "Alice",
  "text":         "hi all",
  "created_at":   "2024-05-13T08:33:20Z",
  "tags":         ["slack", "#general"]
}

The connector clips message text to about 4000 characters. Extra text is cut off and gets the suffix …[clipped]. Thread replies set kind to thread_reply, carry the parent's thread_ts, and add a "thread" tag.

Reference

The package has three small modules. slack_sync.sync holds config, record building, the sync loop, and the CLI. slack_sync.api is the Slack Web API client. slack_sync.state is the EvolutionDB memory backend.

slack_sync.sync

def main(argv: Optional[list] = None) -> int

The CLI entry point. It is the evolutiondb-slack-sync console script. It parses arguments, loads the dotenv file, builds Config, then runs one pass or loops on --interval. It returns a process exit code. The code is 2 for a bad --since or a missing token. The code is 4 if a pass raises. Otherwise the code is 0.

def sync_once(cfg: Config, *, since_default: Optional[float], dry_run: bool = False) -> Dict[str, int]

Runs a single sync pass. It builds a SlackClient. It also builds a MemoryStore unless dry_run is set. It loops over every conversation, saves messages and thread replies, and advances watermarks. It returns the counter dict with channels, messages, replies, skipped, and errors. A per-channel SlackError is logged to stderr and counted as an error. It does not stop the pass. It raises RuntimeError if the token is empty.

def parse_since(text: str) -> float

Parses a <n><s|m|h|d|w> duration into a Unix-seconds float. The value is relative to now in UTC. It raises ValueError on anything else.

class Config

Reads all environment variables into typed attributes. The attributes are token, evosql_host, evosql_port, evosql_user, evosql_pass, evosql_db, user_id, store, and poll_secs. The CLI builds it after loading the dotenv file.

There are some internal helpers. _build_record(...) builds the record dict shown above. _ts_to_iso(ts) converts a Slack ts to an ISO-8601 Z string, or to None. _short(s, n=4000) clips long text. _load_dotenv(path) fills the environment without overwriting existing values.

slack_sync.api

class SlackClient(token: str, user_agent: str = "evolutiondb-slack-sync")

A small Slack Web API client built on urllib only. It authenticates every request with Authorization: Bearer <token>. On an HTTP 429 it follows the Retry-After header and retries exactly once. Any other failed response raises SlackError.

SlackClient.list_conversations() -> Iterator[Dict]

Cursor-paginates conversations.list across all four conversation types. It leaves out archived ones. It yields one conversation dict at a time.

SlackClient.history(channel: str, oldest: Optional[float] = None) -> Iterator[Dict]

Pages conversations.history for one channel and buffers the batch. It yields messages sorted by ts in time order. This keeps the watermark logic monotonic.

SlackClient.replies(channel: str, thread_ts: str, oldest: Optional[float] = None) -> Iterator[Dict]

Pages conversations.replies for one thread. It skips the parent message because history already captured it. It yields the replies in time order.

SlackClient.user_name(user_id: str) -> str

Resolves a U… id to a display name via users.info. It uses an in-memory cache. It falls back through display_name, then real_name, then name, then the raw id. It returns the id itself if the lookup fails. This can happen for a deleted user or a missing scope.

def conversation_display_name(conv: Dict, client: SlackClient) -> str

Makes a human label for a conversation row. A DM resolves to the other user's display name. A group DM resolves to its purpose.value. A channel resolves to #name.

class SlackError(code: str, msg: str, url: str)

Raised for non-ok Slack responses and HTTP errors. It exposes .code and .url. Common .code values include not_in_channel, channel_not_found, and missing_scope.

slack_sync.state

class MemoryStore(host, port, user, password, database, store, namespace)

The EvolutionDB memory backend. It connects via psycopg in autocommit mode. On init it tries to run CREATE MEMORY STORE <store>. All operations run through a small reconnect-with-backoff wrapper. The wrapper tries 3 times and recovers from dropped connections. Each connector keeps its own MemoryStore. So the packages do not depend on one another.

MemoryStore.get_watermark(channel: str) -> Optional[str]

Reads the stored Slack ts watermark for a channel. The key is slack_state_<channel>. It returns None if the channel has not been seen yet.

MemoryStore.set_watermark(channel: str, ts: str) -> None

Writes the channel watermark with MEMORY PUT INTO <store>. It stores the Slack ts exactly as is. The round-trip keeps microsecond precision.

MemoryStore.put_record(key: str, record: Dict) -> None

Serializes a record dict to JSON. It writes the JSON with MEMORY PUT INTO <store> under the configured namespace.

Notes and limitations

  • Beta. The package is version 0.1.0 and marked Development Status 4 - Beta. There is no slack_sdk dependency. The API client is hand-rolled on urllib.
  • Read-only by design. The scopes above grant no write access. There is no posting or sending code in this package. The opt-in reply path lives in the outbox action loop. See ADR-004. It also needs chat:write.
  • Rate limits. Slack throttles conversations.history at Tier 3. That is 50+ requests per minute per workspace. The client retries a 429 exactly once after it sleeps for Retry-After. For very busy workspaces, raise SLACK_POLL_INTERVAL instead of looping tighter.
  • SQL building uses string interpolation. state.py escapes values with a local _e() helper. The helper strips newlines and tabs and doubles single quotes. It then puts the values into the MEMORY PUT and SELECT statements instead of using parameter binding.
  • Privacy. The user-scope token reads only the conversations the human can already see. There is no privilege escalation. But once a message lands in EvolutionDB, anyone with the matching MCP_USER_ID namespace can recall it via search_memory. Make sure it is okay to pipe work Slack into a personal memory store before you turn it on.
  • Tests. tests/test_sync.py covers the pure functions only. These are parse_since, _ts_to_iso, and _build_record. The package has no integration tests against a live Slack workspace or database.
