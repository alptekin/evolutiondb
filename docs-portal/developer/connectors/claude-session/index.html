Claude sessions connector

The connector reads your Claude Code session transcripts and saves them into EvolutionDB long-term memory. The assistant can then search your past conversations. It reads local files only. It needs no API and no token.

What it does

Claude Code writes JSONL transcript files under ~/.claude/projects/<slug>/<sessionId>.jsonl. The package evolutiondb-claude-session-sync reads these files. It turns each session into one searchable digest record in EvolutionDB memory. After a session is indexed, you can find its digest with the MCP server's search_memory tool. For example, you can ask "what did we decide about the storage migration last week?".

Each session creates two memory keys.

  • claude_session_<sessionId> holds the digest. The digest has an AI title when one is present. It also has trimmed user and assistant turns, the project, the git branch, the turn count, and the start and end timestamps.
  • claude_session_state_<sessionId> is a watermark for one session. It holds the last seen timestamp.

The keys are deterministic. If you re-sync a session that has grown, the connector upserts the same record. It does not make a duplicate. The connector keeps only the real conversation. It keeps real user prompts. It filters out slash-command, hook, and caveat wrappers and tool_result blocks. It keeps the assistant's prose. It skips tool calls and thinking blocks. The digest is clipped so it stays under the engine's value cap.

Install

# editable install from the repo
pip install -e client/claude-session-sync

You need Python >=3.9. The only runtime dependency is psycopg[binary]>=3.1. The connector talks to EvolutionDB over the PostgreSQL wire protocol. PII masking is optional. See Notes.

Configure with env vars and auth

You do not need any secrets. The connector reads local transcript files. It reads the connection and namespace settings from environment variables. It loads a .env file from the working directory on its own. You can pass --env-file to use a file in another place. The shipped .env.example lists every variable. 

# Where Claude Code stores session transcripts (one JSONL per session)

Usage

The installed entry script is evolutiondb-claude-session-sync.

evolutiondb-claude-session-sync --once             # one pass
evolutiondb-claude-session-sync --once --dry-run   # parse, persist nothing
evolutiondb-claude-session-sync --interval 900     # daemon
evolutiondb-claude-session-sync --dir ~/.claude/projects/<one-project>

Each pass prints one JSON line to stdout. For example, it prints {"ok": true, "sessions": N, "skipped": N, "errors": N}. If the pass fails, it prints {"ok": false, "error": "…"}. The exit code is 0 when the pass is ok. The exit code is 4 when the pass fails.

The CLI accepts these flags.

FlagEffect
--onceRun a single pass and exit.
--interval NRun in daemon mode. It repeats every N seconds. The value is floored to 60.
--dir PATHOverride CLAUDE_PROJECTS_DIR for this run.
--dry-runParse transcripts but save nothing. It opens no DB connection.
--env-file PATHPath to the dotenv file. It defaults to .env. It is ignored if the file is missing.

If you give neither --once nor --interval, the connector runs a single pass. This is the same as --once. In daemon mode it installs SIGTERM and SIGINT handlers. These handlers finish the current pass before they stop.

Reference

The public surface lives in claude_session_sync.sync and claude_session_sync.state.

def parse_session(path: Path) -> Optional[Dict]

parse_session parses one transcript file into a session record. It returns None if the file has no usable conversation. A file is not usable when it has no session id, or when it has zero user and assistant turns. It reads type=user lines and keeps real string prompts only. It reads type=assistant lines and keeps text blocks only. It picks up aiTitle from an ai-title line. It leaves out sidechain lines. Sidechain lines are subagent lines. The returned dict includes these fields. fact, source with the value "claude_session", kind with the value "session", session_id, title, project, cwd, git_branch, turns, text for the digest, started_at, ended_at, and tags.

def sync_once(cfg: Config, *, dry_run: bool = False) -> Dict[str, int]

sync_once globs **/*.jsonl under the configured projects dir. It parses each file. It upserts sessions that are new or have grown. It skips a session when its ended_at is at or before the stored watermark. It returns the counters {"sessions", "skipped", "errors"}. It raises RuntimeError if the projects dir does not exist. When dry_run=True, it never opens a DB connection.

class Config

Config reads all settings from environment variables. See Configure. The settings are projects_dir, evosql_host, evosql_port, evosql_user, evosql_pass, evosql_db, user_id from MCP_USER_ID, store, and poll_secs.

def main() -> int

main is the CLI entry point. It is registered as the evolutiondb-claude-session-sync console script. It parses the args. It loads the dotenv file. It builds a Config. It installs signal handlers. It runs once or loops on the interval. It returns the process exit code.

class MemoryStore(host, port, user, password, database, store, namespace)

MemoryStore is the DB backend in claude_session_sync.state. It connects with psycopg in autocommit mode. It issues CREATE MEMORY STORE. It ignores the failure if the store already exists. It reconnects up to 3 times with backoff when a connection drops. It has these methods.

  • get_watermark(session_id) -> Optional[str] reads the stored updated_at for a session from __mem_<store>.
  • set_watermark(session_id, updated_at) writes the claude_session_state_<sessionId> row with MEMORY PUT INTO.
  • put_record(key, record) encodes the record as JSON and writes it with MEMORY PUT INTO.

Notes and limitations

  • Local files only. The connector never calls an API. It reads the transcript JSONL files Claude Code already writes on disk.
  • The digest has a limit. The digest text is capped at 5000 characters. The cap is an internal value named _DIGEST_CAP. When the text is cut, the connector adds …[clipped] at the end. This keeps the record JSON under the engine's value cap of about 8 KB.
  • Watermark granularity. A session is re-indexed only when its latest turn timestamp is newer than the stored watermark. Sessions that are equal or older count as skipped.
  • Privacy and cleartext. Transcripts are real work. Your prompts, code, and assistant output land in EvolutionDB in cleartext under your namespace. Anyone with DB access and the namespace can find them with search_memory. You can limit what gets indexed. Point --dir or CLAUDE_PROJECTS_DIR at a single project subfolder.
  • Optional PII masking. If evolutiondb-pii is installed, the connector runs the fact, title, and text fields through its masker. It then writes encrypted companion rows. The key prefix is <key>_pii. If it is not installed, records are written without masking.
  • Status. The version is 0.1.0 and it is classified as Beta. The included tests are in tests/test_sync.py. They cover only parsing and extraction with hand-built fixtures. They do not touch the database.

See the MCP server to learn how these records are recalled. See the project wiki to learn the memory-store internals.
