Claude sessions connector

Index your Claude Code session transcripts into EvolutionDB long-term memory so the assistant can search past conversations — a local-file connector, no API and no token.

What it does

The package evolutiondb-claude-session-sync reads the JSONL transcript files Claude Code writes under ~/.claude/projects/<slug>/<sessionId>.jsonl and turns each session into one searchable digest record in EvolutionDB memory. Once indexed, the digests are recallable through the MCP server's search_memory tool (e.g. "what did we decide about the storage migration last week?").

Each session produces two memory keys:

  • claude_session_<sessionId> — the digest: an AI title (when present), trimmed user/assistant turns, project, git branch, turn count, and start/end timestamps.
  • claude_session_state_<sessionId> — a per-session watermark holding the last seen timestamp.

Keys are deterministic, so re-syncing a session that grew upserts the same record instead of duplicating it. The connector keeps only genuine conversation: real user prompts (slash-command, hook, caveat wrappers and tool_result blocks are filtered out) and the assistant's prose (tool calls and thinking blocks are skipped). The digest is clipped to stay under the engine's value cap.

Install

# editable install from the repo
pip install -e client/claude-session-sync

Requires Python >=3.9. The only runtime dependency is psycopg[binary]>=3.1 (the connector talks to EvolutionDB over the PostgreSQL wire protocol). PII masking is optional — see Notes.

Configure (env vars / auth)

No secrets are required — the connector reads local transcript files. Connection and namespace settings are read from environment variables (a .env file in the working directory is loaded automatically; pass --env-file to point elsewhere). The shipped .env.example lists every variable: 

# Where Claude Code stores session transcripts (one JSONL per session)
CLAUDE_PROJECTS_DIR=~/.claude/projects

# --- EvolutionDB connection ---
EVOSQL_HOST=127.0.0.1
EVOSQL_PORT=5433
EVOSQL_USER=admin
EVOSQL_PASSWORD=admin
EVOSQL_DATABASE=evosql

# Namespace shared with the MCP server (its MCP_USER_ID)
MCP_USER_ID=default_user

# Memory store the session records land in (the MCP server's primary store)
CLAUDE_SESSION_STORE=mcp_mem

Usage

The installed entry script is evolutiondb-claude-session-sync:

evolutiondb-claude-session-sync --once             # one pass
evolutiondb-claude-session-sync --once --dry-run   # parse, persist nothing
evolutiondb-claude-session-sync --interval 900     # daemon
evolutiondb-claude-session-sync --dir ~/.claude/projects/<one-project>

Each pass prints one JSON line to stdout, for example {"ok": true, "sessions": N, "skipped": N, "errors": N}. On failure it prints {"ok": false, "error": "…"}. Exit codes: 0 ok · 4 pass failed.

Flags accepted by the CLI:

FlagEffect
--onceRun a single pass and exit.
--interval NDaemon mode: repeat every N seconds (floored to 60).
--dir PATHOverride CLAUDE_PROJECTS_DIR for this run.
--dry-runParse transcripts but persist nothing (no DB connection).
--env-file PATHDotenv path; defaults to .env, ignored if missing.

With neither --once nor --interval, the connector runs a single pass (same as --once). In daemon mode it installs SIGTERM/SIGINT handlers that finish the current pass before stopping.

Reference

Public surface lives in claude_session_sync.sync and claude_session_sync.state.

def parse_session(path: Path) -> Optional[Dict]

Parses one transcript file into a session record, or returns None if the file has no usable conversation (no session id, or zero user and assistant turns). It reads type=user lines (genuine string prompts only), type=assistant lines (text blocks only), and picks up aiTitle from an ai-title line. Sidechain (subagent) lines are excluded. The returned dict includes fact, source ("claude_session"), kind ("session"), session_id, title, project, cwd, git_branch, turns, text (the digest), started_at, ended_at, and tags.

def sync_once(cfg: Config, *, dry_run: bool = False) -> Dict[str, int]

Globs **/*.jsonl under the configured projects dir, parses each, and upserts new/grown sessions. Skips a session when its ended_at is at or before the stored watermark. Returns counters {"sessions", "skipped", "errors"}. Raises RuntimeError if the projects dir does not exist. When dry_run=True it never opens a DB connection.

class Config

Reads all settings from environment variables (see Configure): projects_dir, evosql_host, evosql_port, evosql_user, evosql_pass, evosql_db, user_id (from MCP_USER_ID), store, and poll_secs.

def main() -> int

CLI entry point (registered as the evolutiondb-claude-session-sync console script). Parses args, loads the dotenv file, builds a Config, installs signal handlers, and runs once or loops on the interval. Returns the process exit code.

class MemoryStore(host, port, user, password, database, store, namespace)

The DB backend in claude_session_sync.state. Connects with psycopg in autocommit mode and issues CREATE MEMORY STORE (ignoring failure if it already exists). It auto-reconnects up to 3 times with backoff on dropped connections. Methods:

  • get_watermark(session_id) -> Optional[str] — reads the stored updated_at for a session from __mem_<store>.
  • set_watermark(session_id, updated_at) — writes the claude_session_state_<sessionId> row via MEMORY PUT INTO.
  • put_record(key, record) — JSON-encodes the record and writes it with MEMORY PUT INTO.

Notes & limitations

  • Local files only. The connector never calls an API; it reads the transcript JSONL files Claude Code already writes on disk.
  • Digest is bounded. The digest text is capped at 5000 characters (an internal _DIGEST_CAP) and suffixed with …[clipped] when truncated, to keep the record JSON under the engine's ~8 KB value cap.
  • Watermark granularity. A session is re-indexed only when its latest turn timestamp is newer than the stored watermark; equal-or-older sessions count as skipped.
  • Privacy / cleartext. Transcripts are real work — prompts, code, and assistant output land in EvolutionDB in cleartext under your namespace. Anyone with DB access and the namespace can recall them via search_memory. Scope what gets indexed by pointing --dir / CLAUDE_PROJECTS_DIR at a single project subfolder.
  • Optional PII masking. If evolutiondb-pii is installed, the connector runs the fact, title, and text fields through its masker and writes encrypted companion rows (key prefix <key>_pii). When it is not installed, records are written unmasked.
  • Status. Version 0.1.0, classified Beta. The included tests (tests/test_sync.py) cover pure parsing/extraction with hand-built fixtures and do not touch the database.

See also the MCP server for how these records are recalled, and the project wiki for memory-store internals.
