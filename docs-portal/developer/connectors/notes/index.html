Apple Notes connector

This is a macOS-only *-sync connector. It reads your Apple Notes through the JavaScript-for-Automation bridge. It stores each note in EvolutionDB long-term memory. An AI assistant can then recall the note later through search_memory.

What it does

The package evolutiondb-notes-sync talks to Notes.app over the macOS JavaScript-for-Automation (JXA) bridge. It converts each note's HTML body to plain text. Then it saves the note into an EvolutionDB MCP memory store. The MCP server shows those notes through search_memory. An assistant can then answer questions about something you wrote weeks ago.

The connector reads notes through JXA on purpose. It does not parse NoteStore.sqlite directly. The source comments explain why. Apple encodes each note body as gzip plus a private "NotesBody" protobuf inside ZICCLOUDSYNCINGOBJECT.ZDATA. That format changes across macOS releases. JXA returns the already-rendered HTML body and stays stable across versions. The cost is that the user must grant Automation permission. You set this under System Settings → Privacy & Security → Automation → Terminal/iTerm → Notes. macOS asks for this on the first run.

Each pass does the following.

  • Starts /usr/bin/osascript -l JavaScript with an embedded JXA script. The script loops over Application('Notes').notes() and prints one JSON object per note. Each object holds the id, name, body, folder, modified, and created fields.
  • Converts each note's HTML body to plain text. It strips tags, decodes entities, and collapses whitespace.
  • Skips empty notes. An empty note has no text and an untitled name. These count as skipped.
  • Writes one record per note. The key is note_<note-id> with slashes in the id replaced by underscores. The key is always the same for a note. So re-syncing the same note overwrites it instead of making a copy.
  • Moves forward a single ISO-8601 watermark. This is the highest modified_at it has seen. The next pass then only imports notes changed more recently.

Records keep the note title, folder, note id, and the created and modified timestamps. This gives later search_memory queries some context.

What is not stored

  • Notes whose body is empty and whose name is blank. The record builder returns None for these. They count as skipped.
  • Attachments, images, tables, and rich formatting. The body becomes plain text through a regex-based _html_to_text. That function only knows simple tags. It handles h1–h6, p, div, br, and li.
  • Note text past about 3000 characters. The body is clipped and the record is tagged truncated. See Record shape.

Install

The distribution name is evolutiondb-notes-sync. The importable package is notes_sync. It needs Python 3.9+ and runs on macOS only. It depends on psycopg[binary]>=3.1 and evolutiondb-pii>=0.1.1.

# From PyPI
pip install evolutiondb-notes-sync

# Or editable, from this repo
pip install -e client/notes-sync

Installing gives you a single console script named evolutiondb-notes-sync. Its entry point is notes_sync.__main__:main.

The package directory has no README. This page comes from the source code itself and the packaging metadata. The source files are notes_sync/sync.py and notes_sync/state.py.

Configure (env vars / auth)

You configure everything through environment variables. The CLI loads a dotenv file before it reads them. By default it loads .env in the working directory. You can change this with --env-file. Values already in the process environment win. The loader _load_dotenv uses os.environ.setdefault. So it never overwrites a value that is already set. It also skips blank values, comments, and lines without an =.

macOS Automation permission

There is no API key and no OAuth flow. The only auth is the macOS Automation permission. It lets the running terminal control Notes.app. On the first run macOS shows the Automation consent prompt. After you grant it, later runs are silent. You grant it under System Settings → Privacy & Security → Automation → your terminal → Notes. If permission is denied or missing, the notes show up as per-note errors. These errors are logged to stderr.

You may not be on macOS. In that case /usr/bin/osascript is not present. The connector then prints [notes-sync] /usr/bin/osascript not found - Notes sync is macOS-only. and returns nothing.

Usage

The CLI has five flags. They are --once, --interval <seconds>, --since <iso>, --dry-run, and --env-file <path>. The --since flag is empty by default. The --env-file flag is .env by default.

# First run - import everything, then exit.
evolutiondb-notes-sync --once

# Only import notes modified on or after a given ISO timestamp.
evolutiondb-notes-sync --once --since 2026-01-01T00:00:00Z

# Daemon mode: pass, then sleep --interval seconds, repeat.
evolutiondb-notes-sync --interval 600

# Read Notes and build records, but make no DB writes.
evolutiondb-notes-sync --once --dry-run

You can pass --once, or you can leave out --interval. Either way, exactly one pass runs. In daemon mode the interval has a minimum of 60 seconds. A shorter value is raised to 60. SIGTERM and SIGINT ask for a graceful stop after the current pass finishes. The loop checks a _stop flag once per second of sleep.

Each pass prints a one-line JSON summary to stdout. A success line looks like this.

{"ok": true, "notes": 137, "skipped": 4, "errors": 0}

On failure the line is {"ok": false, "error": "<message>"}. The process then exits with code 4. The numbers above are examples. They are not real counts.

--since semantics

--since is an optional ISO-8601 floor. It only sets the lower bound for a first run. An example value is 2026-01-01T00:00:00Z. It is not a relative duration. The value is passed through as is and compared as a string against each note's modified_at. An empty value is the default. It means import everything. Once the store holds a watermark, the watermark wins over --since. The connector uses the stored watermark if there is one. Otherwise it uses the --since value.

Record shape

Each note is stored as a JSON record under the key note_<note-id>. _build_record writes these fields.

{
  "fact":        "Note \"Groceries\": milk, eggs, bread …",  // title + first 240 text chars
  "source":      "notes",
  "kind":        "note",
  "title":       "Groceries",             // "(untitled)" when the note has no name
  "text":        "milk, eggs, bread",     // plain text, clipped to 3000 chars
  "folder":      "Notes",
  "note_id":     "x-coredata://…/ICNote/p42",
  "modified_at": "2026-05-13T08:33:20Z",  // falls back to created when modified is absent
  "created_at":  "2026-05-01T11:00:00Z",
  "truncated":   false,
  "tags":        ["notes", "apple-notes"]  // + "truncated" when text was clipped
}

The note text is clipped to _MAX_TEXT_CHARS = 3000. The source explains the spare room. EvolutionDB caps a single SQL statement at 8KB. The MEMORY PUT wrapper, the JSON envelope, and single-quote doubling can roughly double the byte count on the wire. So 3000 chars leaves room for a note full of quotes or multi-byte UTF-8. When text is clipped, truncated is set to true and a "truncated" tag is added.

PII protection (optional)

evolutiondb-pii may expose protect_record. If it does, the connector runs each record through it before writing. This masks or encrypts the fact, title, and text fields. It uses the key prefix notes_pii_<id>. It can also produce companion records that are written next to the main one. If the import fails, _protect just passes the record through. The record is then stored unchanged. The structural metadata is always stored in the clear. This means the folder, id, and timestamps.

Reference

The package has four small modules. notes_sync.__init__ holds the version and the docstring. notes_sync.__main__ is the script shim that calls main. notes_sync.sync holds the config, the JXA, record building, the sync loop, and the CLI. notes_sync.state is the EvolutionDB memory backend. There is no separate API-client module. The client is the embedded JXA script driven by osascript.

notes_sync.sync

def main(argv: Optional[List[str]] = None) -> int

This is the CLI entry point. It is the evolutiondb-notes-sync console script. It parses arguments, loads the dotenv file, builds Config, and installs signal handlers. Then it runs one pass or loops on --interval. It returns a process exit code. The code is 4 if a pass raises an error. Otherwise it is 0.

def sync_once(cfg: Config, *, since_iso: Optional[str], dry_run: bool = False) -> Dict[str, int]

Runs a single sync pass. If dry_run is off, it builds a MemoryStore and reads its watermark. The floor it uses is watermark or since_iso. It loops over every note through iter_notes. It skips empty notes and notes at or below the floor. It writes the rest, plus any PII companion records. It moves the watermark forward to the highest modified_at it has seen. Then it returns the counter dict with notes, skipped, and errors. A single failing note is logged to stderr and counted as errors. It does not stop the pass.

def iter_notes() -> Iterable[Dict]

Starts /usr/bin/osascript -l JavaScript running the embedded JXA. It streams one JSON note per line. JXA's console.log writes to stderr. So stdout and stderr are merged with stderr=STDOUT. Without this, the stderr pipe would fill up at about 64KB on the first oversized note. Then osascript would block forever. Non-JSON lines are ignored. Lines with an error key are logged and skipped. If osascript is missing, you are not on macOS. It then prints a notice and returns nothing.

def _html_to_text(body_html: str) -> str

A cheap regex-based HTML-to-text conversion. It turns the block tags br, p, div, h1–h6, and li into newlines. It strips all other tags. It decodes HTML entities and collapses whitespace. It is not a full HTML parser. It is good enough for the simple markup that Apple Notes emits.

class Config

Reads all environment variables into typed attributes. The attributes are evosql_host, evosql_port, evosql_user, evosql_pass, evosql_db, user_id, and store. It is built after the dotenv file is loaded.

There are several internal helpers. _build_record(n) builds the record dict shown above. It returns None for an empty note. _note_key(n) builds the fixed note_<id> key. _protect(rec, main_key) applies optional PII masking and returns (record, companions). _load_dotenv(path) fills the environment without overwriting existing values. _install_signal_handlers() wires SIGTERM and SIGINT to a graceful-stop flag.

notes_sync.state

class MemoryStore(host, port, user, password, database, store, namespace)

The EvolutionDB memory backend. It connects through psycopg in autocommit mode. On init it tries to run CREATE MEMORY STORE <store> on a best-effort basis. All operations run through a reconnect-with-backoff wrapper named _retry. It makes 3 attempts. The backoff is 0.5s times the attempt number. It recovers from dropped connections such as OperationalError and InterfaceError. The module docstring says it is a mirror of the sibling connector packages. This keeps each connector free of any dependency on the others.

MemoryStore.get_watermark_iso() -> Optional[str]

Reads the stored ISO watermark for the configured namespace. The key is notes_state_latest. The value is the modified_at field of its JSON value. It returns None if there is no watermark yet.

MemoryStore.set_watermark_iso(iso: str) -> None

Writes the watermark with MEMORY PUT INTO <store>. It stores {"modified_at": iso, "saved_at": <epoch>}.

MemoryStore.put_record(key: str, record: Dict) -> None

Serializes the record to JSON with ensure_ascii=False and writes it under the namespace. It first runs a DELETE for the same key. Then it runs MEMORY PUT INTO <store>. The source comment explains why. MEMORY PUT upserts small payloads fine. But it reports a false "duplicate key" error for larger ones. The cheap pre-delete avoids that path. On failure it logs the key and the value length to stderr and re-raises.

Notes and limitations

  • macOS only. The connector calls out to /usr/bin/osascript and the Notes.app JXA bridge. On any other platform it prints a notice and does nothing.
  • Version mismatch in metadata. The packaging metadata declares version 0.1.2. This is in pyproject.toml and PKG-INFO. But the in-package __version__ in notes_sync/__init__.py is still 0.1.0. It is marked Development Status 4 - Beta.
  • Read-only. No code path writes back to Notes.app. The connector only reads.
  • Plain text only. Bodies are flattened by a regex. It is not a real HTML parser. Attachments, images, tables, and checklists are not kept. Long notes are clipped at 3000 characters and tagged truncated.
  • String-comparison watermark. The incremental floor compares modified_at values as strings. This works only because Notes timestamps are written as fixed-width ISO-8601 UTC. The format is NSISO8601DateFormatWithInternetDateTime. A note with no modification date falls back to its creation date.
  • SQL built by string interpolation. state.py escapes values with a local _e() helper. It strips newlines and tabs and doubles single quotes. It then puts the values into the MEMORY PUT, SELECT, and DELETE statements. It does not use parameter binding.
  • Privacy. Notes can hold very personal data. The optional evolutiondb-pii integration masks fact, title, and text. But that package or a public key may be absent. Then the records are stored in the clear. Anyone with the matching MCP_USER_ID namespace can recall them through search_memory.
  • No tests. The package ships no test suite. There is no tests/ directory in the distribution.
