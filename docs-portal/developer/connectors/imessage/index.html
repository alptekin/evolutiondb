iMessage connector

A local-only, read-only *-sync connector that reads the macOS Messages.app SQLite database (chat.db) directly and pours your iMessage and SMS history into EvolutionDB long-term memory, so an AI assistant can recall it later through search_memory.

What it does

The package evolutiondb-imessage-sync opens the local Messages.app store at ~/Library/Messages/chat.db, reads new messages incrementally, and persists each one into an EvolutionDB MCP memory store. The MCP server then surfaces those messages through search_memory. Unlike the API-based connectors (Slack, Gmail), there is no network API and no OAuth token: the data comes straight off disk, so the connector only runs on the same Mac whose Messages history you want to sync.

Concretely, each pass:

  • Snapshots chat.db (plus its -wal / -shm sidecars) to a temp file with shutil.copy, because Messages.app holds an exclusive WAL lock on the live file.
  • Runs one SQL SELECT joining message → handle → chat_message_join → chat, filtered to m.date > ? and ordered ascending by date.
  • Decodes each row's body, using the plain text column when present, otherwise extracting the string out of the attributedBody blob (see below).
  • Writes one record per message, keyed imessage_<apple_date>_<is_from_me>_<handle>_<chat_id>. The key is deterministic, so re-sync UPSERTs rather than duplicating.
  • Advances a single watermark (the highest Apple-epoch date seen) so the next pass only reads newer messages.

Records carry the service (iMessage/SMS), sender label, handle, chat label, direction, and the message's ISO timestamp, so downstream search_memory queries can filter on them.

Reading the attributedBody blob

From macOS 11 onward, Messages.app silently moved many newer messages: replies, link previews, mentions, SMS link cards, edited variants, out of the plain text column and into attributedBody, leaving text NULL. The connector decodes two formats so those rows don't all surface as (rich message) placeholders:

  • Legacy NSArchiver typedstream (magic \x04\x0bstreamtyped), the format Messages.app still uses for almost every message. The decoder locates the NSString/NSMutableString class tag, then reads typedstream's compact length marker (single byte, 0x81+uint16, or 0x82+uint32) and the UTF-8 payload.
  • NSKeyedArchiver bplist (magic bplist00), used for some rich payloads on newer releases. Parsed via plistlib, resolving the $top.root UID, with a longest-non-class-string fallback.

The module's own docstring notes that roughly 95% of attributedBody messages decode cleanly; the remainder (tap-back reactions, sticker payloads, Apple Pay receipts, etc.) carry no plain-text equivalent and stay as the (rich message) placeholder.

Install

The distribution name is evolutiondb-imessage-sync; the importable package is imessage_sync. It requires Python 3.9+ and depends on psycopg[binary]>=3.1 and evolutiondb-pii>=0.1.1. It is macOS-only (it reads a macOS-specific file path and SQLite schema).

# From PyPI
pip install evolutiondb-imessage-sync

# Or editable, from this repo
pip install -e client/imessage-sync

Installing exposes a single console script, evolutiondb-imessage-sync (entry point imessage_sync.__main__:main).

Full Disk Access (required)

There is no API key and no login. Instead, the shell or process that runs the connector must be granted Full Disk Access, or macOS will deny the read of ~/Library/Messages/chat.db. Grant it under System Settings → Privacy & Security → Full Disk Access for the terminal app (or the launchd/agent process) that invokes the sync. If the snapshot copy raises PermissionError, the connector falls back to opening the live file read-only and immutable (file:…?mode=ro&immutable=1), but Full Disk Access is still required to read it at all.

Configure (env vars / auth)

Configuration is entirely through environment variables. The CLI loads a dotenv file before reading them, by default .env in the working directory, overridable with --env-file. Existing process environment values win (the loader uses setdefault, so it never overwrites what is already set).

Optional PII protection

If the sibling package evolutiondb-pii is importable, each record is passed through its protect_record (detect → mask → encrypt) over the fields fact, text, sender, handle, and chat before it is stored, keyed under <msg_key>_pii. Per the inline comment, with no public key on disk this is a transparent passthrough. If the import fails, protection is skipped entirely (_pii_protect is None and records are stored as-is).

Usage

The CLI has five flags: --once, --interval <seconds>, --since <duration> (default 30d), --dry-run, and --env-file <path> (default .env).

# First run - pull the last 30 days, then exit.
evolutiondb-imessage-sync --once --since 30d

# Daemon mode: pass, then sleep --interval seconds, repeat.
evolutiondb-imessage-sync --interval 600

# Read + decode only - no DB writes, no MemoryStore is constructed.
evolutiondb-imessage-sync --once --dry-run

If --once is given, or no --interval is supplied, exactly one pass runs. In daemon mode the interval is clamped to a minimum of 60 seconds, and SIGTERM/SIGINT request a graceful stop after the current pass finishes (the loop sleeps in 1-second steps and checks the stop flag between them).

Each pass prints a one-line JSON summary to stdout. On success it reports the two counters that the pass tracks:

{"ok": true, "messages": 137, "errors": 0}

On failure the line is {"ok": false, "error": "<message>"}. (The figures above are illustrative, not guaranteed counts.) A read failure inside a pass is caught, logged to stderr, and counted as one error rather than aborting the process.

--since syntax

--since takes a duration of the form <n><unit>, where unit is one of s, m, h, d, w (e.g. 30s, 5m, 1h, 7d, 2w). It is converted to an Apple-epoch nanoseconds floor (Apple counts time from 2001-01-01 UTC), which becomes the lower bound on message.date. A garbage value exits with code 2. Note: --since is only the default floor: once a watermark has been stored, the watermark takes precedence over --since.

Record shape

Each message is stored as a JSON record under the key imessage_<apple_date>_<is_from_me>_<handle>_<chat_id> (spaces stripped from handle and chat_id). Fields written by _build_record:

{
  "fact":        "iMessage from +1555…: see you at 6",
  "source":      "imessage",
  "kind":        "message",
  "service":     "iMessage",        // or "SMS"
  "text":        "see you at 6",
  "sender":      "+1555…",          // "me" when is_from_me
  "handle":      "+1555…",
  "chat":        "Family",
  "chat_id":     "chat123…",
  "is_from_me":  false,
  "modified_at": "2024-05-13T08:33:20Z",
  "apple_date":  734567600000000000,
  "tags":        ["imessage", "chat", "incoming"]
}

The fact field is a one-line human summary and its body is clipped to ~240 characters; the full body is kept in text (clipped to ~200 chars by _short's default elsewhere). Outgoing messages set sender to "me", is_from_me to true, and tag "outgoing"; incoming messages tag "incoming". Rows whose body resolves to empty are skipped entirely; rows with a non-decodable attributedBody are stored with text = "(rich message)".

Reference

The package is four small modules: imessage_sync.sync (config, key building, sync loop, CLI), imessage_sync.extract (chat.db reader + attributedBody decoders), imessage_sync.state (EvolutionDB memory backend), and imessage_sync.__main__ (a one-line wrapper that calls sync.main).

imessage_sync.sync

def main(argv: Optional[List[str]] = None) -> int

CLI entry point (the evolutiondb-imessage-sync console script). Parses arguments, loads the dotenv file, parses --since, builds Config, installs signal handlers, then runs one pass or loops on --interval. Returns a process exit code: 2 for a bad --since, 4 if a pass raises, otherwise the result of the last pass (0 on success).

def sync_once(cfg: Config, *, since_ns_floor: int, dry_run: bool = False) -> dict

Runs a single sync pass. Unless dry_run, constructs a MemoryStore and reads its watermark; the effective floor is the watermark if present, else since_ns_floor. Iterates extract.iter_messages, skips rows at or below the floor, optionally PII-protects each record, writes it (and any PII companion records), and advances the watermark to the highest date seen. Returns the counter dict {"messages": …, "errors": …}. A read exception is caught, logged, and counted as one error.

def parse_since(text: str) -> int

Parses a <n><s|m|h|d|w> duration and returns the corresponding Apple-epoch nanoseconds value (seconds since 2001-01-01 UTC × 1e9). Raises ValueError on anything else.

class Config

Reads all environment variables into typed attributes (evosql_host, evosql_port, evosql_user, evosql_pass, evosql_db, user_id, store, poll_secs). Constructed after the dotenv file is loaded.

Internal helpers: _msg_key(record) builds the deterministic record key; _protect(rec, main_key) applies evolutiondb_pii when available (else returns the record unchanged with no companions); _load_dotenv(path) populates the environment without overwriting existing values; _install_signal_handlers() wires SIGTERM/SIGINT to a graceful stop flag.

imessage_sync.extract

def iter_messages(since_apple_date: int = 0) -> Iterator[Dict]

Yields normalised message records newer than since_apple_date (Apple's nanoseconds-since-2001 epoch; pass 0 to bootstrap). Locates chat.db, snapshots and connects, runs the join query, converts each date to ISO, builds a record, and yields the non-None ones. If chat.db is missing or a SQLite error occurs, it logs to stderr and returns without yielding.

def chat_db_path() -> Path

Returns the default store path, ~/Library/Messages/chat.db (expanded).

def _decode_attributed_body(blob: Optional[bytes]) -> str

Dispatches an attributedBody blob to the right decoder by magic bytes: _decode_typedstream for \x04\x0bstreamtyped, _decode_keyed_archive for bplist00. Returns the plain string, or "" when neither path finds a body (callers then fall back to the (rich message) placeholder).

def _decode_typedstream(blob: bytes) -> str

Extracts the NSString payload out of Apple's legacy NSArchiver typedstream serialisation, decoding the compact length marker and UTF-8 bytes.

def _decode_keyed_archive(blob: bytes) -> str

Parses an NSKeyedArchiver bplist00 blob via plistlib, resolving the root object's string (with a longest-non-class-string fallback).

Internal helpers: _snapshot(src) copies chat.db + -wal/-shm to a temp file; _connect(src) connects to the snapshot, falling back to a read-only immutable URI on PermissionError; _apple_to_iso(date_value) converts an Apple date to an ISO-8601 Z string (guarding nanoseconds-vs-seconds by magnitude, values above 1e12 are treated as nanoseconds); _build_record(row, iso) assembles the record dict shown above; _short(s, n=200) clips long text with an ellipsis.

imessage_sync.state

class MemoryStore(host, port, user, password, database, store, namespace)

The EvolutionDB memory backend, described in its own docstring as a "mirror of the sibling connector packages." Connects via psycopg (autocommit) and best-effort issues CREATE MEMORY STORE <store> on init (errors swallowed). All operations run through a reconnect-with-backoff wrapper (_retry): up to 3 attempts with a 0.5s × attempt backoff, recovering only from OperationalError/InterfaceError. If psycopg is not installed, the constructor raises RuntimeError with an install hint.

MemoryStore.get_watermark() -> Optional[int]

Reads the single stored watermark (key imessage_state_latest) for the namespace and returns its apple_date_ns integer, or None if not yet seen. Note this is a single global watermark, not per-chat.

MemoryStore.set_watermark(apple_date_ns: int) -> None

Writes the watermark as JSON ({"apple_date_ns": …, "saved_at": …}) with MEMORY PUT INTO <store> under the configured namespace.

MemoryStore.put_record(key: str, record: Dict) -> None

Serializes a record dict to JSON (ensure_ascii=False) and writes it with MEMORY PUT INTO <store> under the namespace.

Notes and limitations

  • Beta, and the version metadata is inconsistent. pyproject.toml declares version 0.1.2 (Development Status 4 - Beta), while imessage_sync/__init__.py still hard-codes __version__ = "0.1.0". There is no README shipped with the package; this page is sourced directly from the code and packaging metadata.
  • macOS-only and local-only. There is no API and no token. The connector reads ~/Library/Messages/chat.db on the same machine, so it must run on the Mac whose history you want, with Full Disk Access granted.
  • Read-only. The package only reads chat.db and writes to EvolutionDB. There is no code that sends iMessages.
  • Single watermark, not per-chat. Unlike the Slack connector's per-channel watermark, this one stores one global high-water mark (imessage_state_latest), the maximum Apple-date seen across all messages.
  • IMESSAGE_POLL_INTERVAL is read but not used for the loop. Config.poll_secs reads it, but main's daemon loop sleeps on the --interval CLI flag instead. To run on a schedule, pass --interval (or wrap a --once run in cron/launchd).
  • Not all rich messages decode. Per extract.py, ~95% of attributedBody messages decode to plain text; tap-backs, stickers, Apple Pay receipts and similar carry no plain-text body and are stored as "(rich message)".
  • SQL building uses string interpolation. state.py escapes values with a local _e() helper (newline/tab stripping + single-quote doubling) and interpolates them into the MEMORY PUT / SELECT statements rather than using parameter binding.
  • Privacy. This pipes personal message history into a memory store. Optional evolutiondb-pii protection is a transparent passthrough unless a key is configured on disk, and once a message lands in EvolutionDB anyone with the matching MCP_USER_ID namespace can recall it via search_memory. Confirm this is acceptable before enabling.
  • Tests. No test files are shipped inside this package.
