iMessage connector

This is a local-only, read-only *-sync connector. It reads the macOS Messages.app SQLite database chat.db straight from disk. It then saves your iMessage and SMS history into EvolutionDB long-term memory. An AI assistant can recall it later through search_memory.

What it does

The package evolutiondb-imessage-sync opens the local Messages.app store at ~/Library/Messages/chat.db. It reads new messages a few at a time. It saves each one into an EvolutionDB MCP memory store. The MCP server then shows those messages through search_memory. The Slack and Gmail connectors use a network API. This one does not. There is no network API and no OAuth token. The data comes straight off disk. So the connector only runs on the same Mac that holds the Messages history you want to sync.

Each pass does this work:

  • Copies chat.db and its -wal and -shm sidecar files to a temp file with shutil.copy. This is needed because Messages.app holds an exclusive WAL lock on the live file.
  • Runs one SQL SELECT. It joins message → handle → chat_message_join → chat. It filters on m.date > ? and sorts by date in ascending order.
  • Decodes each row's body. It uses the plain text column when that column has a value. If not, it pulls the string out of the attributedBody blob. See below for more.
  • Writes one record per message. The key is imessage_<apple_date>_<is_from_me>_<handle>_<chat_id>. The key is always the same for the same message. So a re-sync UPSERTs the record instead of making a copy.
  • Moves a single watermark forward. The watermark is the highest Apple-epoch date seen. So the next pass only reads newer messages.

Each record holds the service such as iMessage or SMS. It also holds the sender label, the handle, the chat label, the direction, and the message's ISO timestamp. So later search_memory queries can filter on these fields.

Reading the attributedBody blob

From macOS 11 onward, Messages.app quietly moved many newer messages out of the plain text column. These include replies, link previews, mentions, SMS link cards, and edited messages. They now live in attributedBody, and text is NULL. The connector decodes two formats. This stops those rows from all showing up as (rich message) placeholders.

  • Legacy NSArchiver typedstream has the magic bytes \x04\x0bstreamtyped. Messages.app still uses this format for almost every message. The decoder finds the NSString or NSMutableString class tag. It then reads the typedstream compact length marker. The marker is a single byte, or 0x81+uint16, or 0x82+uint32. After that it reads the UTF-8 payload.
  • NSKeyedArchiver bplist has the magic bytes bplist00. Some rich payloads on newer releases use this format. It is parsed with plistlib. The decoder resolves the $top.root UID and falls back to the longest non-class string.

The module docstring says that about 95% of attributedBody messages decode cleanly. The rest have no plain-text version. These include tap-back reactions, sticker payloads, and Apple Pay receipts. They stay as the (rich message) placeholder.

Install

The distribution name is evolutiondb-imessage-sync. The package you import is imessage_sync. It needs Python 3.9+. It depends on psycopg[binary]>=3.1 and evolutiondb-pii>=0.1.1. It runs on macOS only. It reads a macOS file path and SQLite schema.

# From PyPI
pip install evolutiondb-imessage-sync

# Or editable, from this repo
pip install -e client/imessage-sync

Installing adds one console script named evolutiondb-imessage-sync. Its entry point is imessage_sync.__main__:main.

Full Disk Access (required)

There is no API key and no login. The shell or process that runs the connector must have Full Disk Access. Without it, macOS blocks the read of ~/Library/Messages/chat.db. Grant it under System Settings → Privacy & Security → Full Disk Access. Grant it to the terminal app or the launchd agent process that runs the sync. The snapshot copy may raise PermissionError. If it does, the connector opens the live file read-only and immutable with file:…?mode=ro&immutable=1. You still need Full Disk Access to read it at all.

Configure (env vars / auth)

You configure the connector with environment variables only. The CLI loads a dotenv file before it reads them. By default this file is .env in the working directory. You can change the path with --env-file. Values already set in the process environment win. The loader uses setdefault. So it never overwrites a value that is already set.

Optional PII protection

The sibling package evolutiondb-pii may be available to import. If it is, each record passes through its protect_record before it is stored. This step runs detect, then mask, then encrypt. It works over the fields fact, text, sender, handle, and chat. The protected copy is keyed under <msg_key>_pii. The inline comment says that with no public key on disk this is a transparent passthrough. If the import fails, protection is skipped. Then _pii_protect is None and records are stored as they are.

Usage

The CLI has five flags. They are --once, --interval <seconds>, --since <duration> with default 30d, --dry-run, and --env-file <path> with default .env.

# First run - pull the last 30 days, then exit.
evolutiondb-imessage-sync --once --since 30d

# Daemon mode: pass, then sleep --interval seconds, repeat.
evolutiondb-imessage-sync --interval 600

# Read + decode only - no DB writes, no MemoryStore is constructed.
evolutiondb-imessage-sync --once --dry-run

Exactly one pass runs if you give --once or if you do not give --interval. In daemon mode the interval has a minimum of 60 seconds. A smaller value is raised to 60. SIGTERM and SIGINT ask for a clean stop after the current pass finishes. The loop sleeps in 1-second steps and checks the stop flag between them.

Each pass prints a one-line JSON summary to stdout. On success it reports the two counters that the pass tracks.

{"ok": true, "messages": 137, "errors": 0}

On failure the line is {"ok": false, "error": "<message>"}. The numbers above are only examples. They are not real counts. A read failure inside a pass is caught and logged to stderr. It is counted as one error. It does not stop the process.

--since syntax

--since takes a duration in the form <n><unit>. The unit is one of s, m, h, d, w. Some examples are 30s, 5m, 1h, 7d, and 2w. The value is turned into an Apple-epoch nanoseconds floor. Apple counts time from 2001-01-01 UTC. This floor becomes the lower bound on message.date. A bad value exits with code 2. --since is only the default floor. Once a watermark is stored, the watermark wins over --since.

Record shape

Each message is stored as a JSON record. The key is imessage_<apple_date>_<is_from_me>_<handle>_<chat_id>. Spaces are stripped from handle and chat_id. These are the fields that _build_record writes.

{
  "fact":        "iMessage from +1555…: see you at 6",
  "source":      "imessage",
  "kind":        "message",
  "service":     "iMessage",        // or "SMS"
  "text":        "see you at 6",
  "sender":      "+1555…",          // "me" when is_from_me
  "handle":      "+1555…",
  "chat":        "Family",
  "chat_id":     "chat123…",
  "is_from_me":  false,
  "modified_at": "2024-05-13T08:33:20Z",
  "apple_date":  734567600000000000,
  "tags":        ["imessage", "chat", "incoming"]
}

The fact field is a one-line summary for people to read. Its body is cut to about 240 characters. The full body is kept in text. That body is cut to about 200 characters by the _short default used elsewhere. Outgoing messages set sender to "me", set is_from_me to true, and add the tag "outgoing". Incoming messages add the tag "incoming". Rows with an empty body are skipped. Rows with an attributedBody that does not decode are stored with text = "(rich message)".

Reference

The package has four small modules. imessage_sync.sync holds the config, key building, sync loop, and CLI. imessage_sync.extract holds the chat.db reader and the attributedBody decoders. imessage_sync.state is the EvolutionDB memory backend. imessage_sync.__main__ is a one-line wrapper that calls sync.main.

imessage_sync.sync

def main(argv: Optional[List[str]] = None) -> int

This is the CLI entry point. It is the evolutiondb-imessage-sync console script. It parses arguments, loads the dotenv file, parses --since, builds Config, and installs signal handlers. It then runs one pass or loops on --interval. It returns a process exit code. The code is 2 for a bad --since. The code is 4 if a pass raises an error. Otherwise it returns the result of the last pass, which is 0 on success.

def sync_once(cfg: Config, *, since_ns_floor: int, dry_run: bool = False) -> dict

Runs a single sync pass. When dry_run is off, it builds a MemoryStore and reads its watermark. The floor is the watermark if there is one. If not, the floor is since_ns_floor. It loops over extract.iter_messages. It skips rows at or below the floor. It can PII-protect each record. It writes the record and any PII companion records. It then moves the watermark to the highest date seen. It returns the counter dict {"messages": …, "errors": …}. A read error is caught and logged. It is counted as one error.

def parse_since(text: str) -> int

Parses a <n><s|m|h|d|w> duration. It returns the matching Apple-epoch nanoseconds value. That value is seconds since 2001-01-01 UTC times 1e9. It raises ValueError on anything else.

class Config

Reads all environment variables into typed attributes. These are evosql_host, evosql_port, evosql_user, evosql_pass, evosql_db, user_id, store, and poll_secs. It is built after the dotenv file is loaded.

Internal helpers. _msg_key(record) builds the stable record key. _protect(rec, main_key) applies evolutiondb_pii when it is available. If it is not, it returns the record unchanged with no companions. _load_dotenv(path) fills the environment without overwriting values that are already set. _install_signal_handlers() wires SIGTERM and SIGINT to a clean stop flag.

imessage_sync.extract

def iter_messages(since_apple_date: int = 0) -> Iterator[Dict]

Yields normalised message records newer than since_apple_date. That value is Apple's nanoseconds-since-2001 epoch. Pass 0 to start from the beginning. It finds chat.db, copies it, and connects. It runs the join query. It turns each date into ISO. It builds a record and yields the records that are not None. If chat.db is missing or a SQLite error happens, it logs to stderr and returns without yielding.

def chat_db_path() -> Path

Returns the default store path. It is the expanded form of ~/Library/Messages/chat.db.

def _decode_attributed_body(blob: Optional[bytes]) -> str

Sends an attributedBody blob to the right decoder based on its magic bytes. It uses _decode_typedstream for \x04\x0bstreamtyped. It uses _decode_keyed_archive for bplist00. It returns the plain string. It returns "" when neither path finds a body. In that case callers fall back to the (rich message) placeholder.

def _decode_typedstream(blob: bytes) -> str

Pulls the NSString payload out of Apple's legacy NSArchiver typedstream format. It decodes the compact length marker and the UTF-8 bytes.

def _decode_keyed_archive(blob: bytes) -> str

Parses an NSKeyedArchiver bplist00 blob with plistlib. It resolves the root object's string and falls back to the longest non-class string.

Internal helpers. _snapshot(src) copies chat.db and its -wal and -shm files to a temp file. _connect(src) connects to the copy. It falls back to a read-only immutable URI on PermissionError. _apple_to_iso(date_value) turns an Apple date into an ISO-8601 Z string. It tells nanoseconds from seconds by size. Values above 1e12 are treated as nanoseconds. _build_record(row, iso) builds the record dict shown above. _short(s, n=200) cuts long text and adds an ellipsis.

imessage_sync.state

class MemoryStore(host, port, user, password, database, store, namespace)

The EvolutionDB memory backend. Its own docstring calls it a "mirror of the sibling connector packages." It connects with psycopg in autocommit mode. On init it tries to run CREATE MEMORY STORE <store> and ignores any error. All operations run through a reconnect-with-backoff wrapper named _retry. It makes up to 3 attempts. The backoff is 0.5s times the attempt number. It only recovers from OperationalError and InterfaceError. If psycopg is not installed, the constructor raises RuntimeError with an install hint.

MemoryStore.get_watermark() -> Optional[int]

Reads the single stored watermark for the namespace. The key is imessage_state_latest. It returns its apple_date_ns integer. It returns None if there is no watermark yet. This is one global watermark. It is not per-chat.

MemoryStore.set_watermark(apple_date_ns: int) -> None

Writes the watermark as JSON in the form {"apple_date_ns": …, "saved_at": …}. It uses MEMORY PUT INTO <store> under the configured namespace.

MemoryStore.put_record(key: str, record: Dict) -> None

Turns a record dict into JSON with ensure_ascii=False. It writes it with MEMORY PUT INTO <store> under the namespace.

Notes and limitations

  • Beta, and the version numbers do not match. pyproject.toml says version 0.1.2 with Development Status 4 - Beta. But imessage_sync/__init__.py still hard-codes __version__ = "0.1.0". The package ships no README. This page comes straight from the code and the packaging metadata.
  • macOS-only and local-only. There is no API and no token. The connector reads ~/Library/Messages/chat.db on the same machine. So it must run on the Mac that holds the history you want. Full Disk Access must be granted.
  • Read-only. The package only reads chat.db and writes to EvolutionDB. No code sends iMessages.
  • Single watermark, not per-chat. The Slack connector keeps a watermark per channel. This one keeps one global high-water mark named imessage_state_latest. It is the highest Apple-date seen across all messages.
  • IMESSAGE_POLL_INTERVAL is read but not used for the loop. Config.poll_secs reads it. But the main daemon loop sleeps on the --interval CLI flag instead. To run on a schedule, pass --interval. You can also wrap a --once run in cron or launchd.
  • Not all rich messages decode. Per extract.py, about 95% of attributedBody messages decode to plain text. Tap-backs, stickers, Apple Pay receipts, and similar items have no plain-text body. They are stored as "(rich message)".
  • SQL is built with string interpolation. state.py escapes values with a local _e() helper. That helper strips newlines and tabs and doubles single quotes. It then puts the values straight into the MEMORY PUT and SELECT statements. It does not use parameter binding.
  • Privacy. This sends personal message history into a memory store. The optional evolutiondb-pii protection is a transparent passthrough unless a key is set on disk. Once a message is in EvolutionDB, anyone with the matching MCP_USER_ID namespace can recall it with search_memory. Make sure this is acceptable before you turn it on.
  • Tests. No test files ship inside this package.
