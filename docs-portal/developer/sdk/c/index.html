C library (libevosql-memory)

A small C client for the EVO text protocol. It adds high-level wrappers for the agent-memory features that ship with EvolutionDB. Every higher-level binding links against it through the FFI layer. Those bindings include Python ctypes, Go cgo, Rust bindgen, and Node node-ffi-napi.

info
This is libevosql-memory 1.0.0. It ships with EvolutionDB 2.x. The header and ABI are stable. New API is added by appending functions to the bottom of evosql_memory.h. Existing call sites never need to recompile.

What it does

The library wraps the EVO text protocol on TCP port 9967. It has three layers. All of them are declared in a single header, include/evosql_memory.h.

  • Typed agent-memory wrappers. MEMORY STORE is a namespaced key-value store with an optional vector embedding. CHECKPOINT STORE holds LangGraph-compatible checkpoint records.
  • A generic escape hatch. Use evo_exec and evo_query for any SQL the typed wrappers do not cover.
  • Streaming surfaces. evo_subscribe uses LISTEN/NOTIFY. evo_cdc_subscribe reads the CDC JSON-lines stream. Each one runs on its own socket and worker thread.

The build produces a shared library as a .dylib, .so, or .dll file. It also produces a static archive named libevosql-memory.a and a pkg-config file. The source is two C files. src/conn.c handles the connection, queries, typed wrappers, and errors. src/subscribe.c handles the streaming workers.

Install

The fastest way is to download a pre-built artifact for your platform. You can find them on the project's GitHub Releases page. Each release ships five platforms.

  • libevosql-memory-linux-x64-vX.Y.Z.tar.gz, Linux glibc x86_64
  • libevosql-memory-linux-arm64-vX.Y.Z.tar.gz, Linux glibc aarch64
  • libevosql-memory-macos-arm64-vX.Y.Z.tar.gz, macOS 14+ Apple Silicon
  • libevosql-memory-macos-x64-vX.Y.Z.tar.gz, macOS 13+ Intel
  • libevosql-memory-windows-x64-vX.Y.Z.zip, Windows 10/11 x86_64

Each archive untars to a directory. The directory holds the shared library, the static archive, and evosql_memory.h. Check the asset's SHA-256 against the SHA256SUMS file attached to the same release.

# download + unpack a pre-built artifact
curl -L https://github.com/alptekin/evolutiondb/releases/latest/download/libevosql-memory-linux-x64-v3.0.1.tar.gz \
    | tar -xz
cp libevosql-memory-linux-x64/libevosql-memory.so /usr/local/lib/
cp libevosql-memory-linux-x64/evosql_memory.h     /usr/local/include/

Build from source

Run the build from client/libevosql-memory. The Makefile detects the host. It supports Darwin, Linux, MinGW, and MSYS. It then picks the right shared-library extension and link flags. Windows builds use MinGW-w64 and link ws2_32 and pthread.

cd client/libevosql-memory
make                       # build/ shared lib + .a + pkg-config
make test                  # runs against 127.0.0.1:9967 (server must be up)
make install PREFIX=/usr/local

# override toolchain / prefix the usual way
make CC=clang
make PREFIX=/opt/evosql install

The build puts its files in build/. You get the shared library libevosql-memory.{dylib,so,dll}, the static archive libevosql-memory.a, and the evosql-memory.pc pkg-config file. You also get the test binaries test_sdk and test_subscribe.

Compiling your program against it

# link the static archive (no install needed)
cc -Iclient/libevosql-memory/include demo.c \
   client/libevosql-memory/build/libevosql-memory.a -o demo

# …or the shared library after `make install`
cc demo.c -levosql-memory -o demo

The streaming functions need -lpthread on POSIX. On Windows, link ws2_32 and pthread the same way the Makefile does.

Configure (connection & auth)

There are no environment variables or config files. You pass connection details as function arguments. evo_connect() takes host, port, user, and password directly. A stock server uses the default credentials admin/admin. The default EVO port is 9967.

Authentication uses the EVO protocol's plaintext path. The client sends the greeting and declines STARTTLS. It then authenticates with the legacy cleartext AUTH <user> <pass> line. The server accepts this line as a fallback.

client → "EVO\n"
server → "HELLO EvoSQL 1.0\n"
server → "STARTTLS\n"           (optional, server-driven)
client → "NOTLS\n"              (the SDK declines TLS in v1)
server → "AUTH_SCRAM\n"
client → "AUTH <user> <pass>\n"
server → "AUTH_OK\n"
warning
v1 is plaintext only. It has no TLS client, no SCRAM, and no extended-query or batch support. Credentials cross the wire in cleartext. Keep the connection on a trusted network or behind a tunnel.

Usage

Put / get memory

The example below comes from examples/put_get.c. It opens a connection, creates a store, writes JSON values, and reads them back.

#include <stdio.h>
#include <evosql_memory.h>

int main(void)
{
    evo_conn_t *c = evo_connect("127.0.0.1", 9967, "admin", "admin");
    if (!c) {
        fprintf(stderr, "connect failed: %s\n", evo_strerror(NULL));
        return 1;
    }

    evo_exec(c, "DROP MEMORY STORE IF EXISTS demo");
    evo_exec(c, "CREATE MEMORY STORE demo");

    evo_memory_put(c, "demo", "users", "alice",
                   "{\"role\":\"pm\"}", NULL);

    char value[1024];
    if (evo_memory_get(c, "demo", "users", "alice",
                       value, sizeof(value)) == EVO_OK) {
        printf("alice -> %s\n", value);
    }

    evo_close(c);
    return 0;
}

Reactive subscription

The example below comes from examples/reactive_agent.c. evo_subscribe() opens its own connection and runs LISTEN <channel>. It then sends each NOTIFY to your callback on a worker thread.

static void on_event(const char *channel, const char *payload,
                     uint64_t lsn, void *ctx)
{
    (void)lsn; (void)ctx;
    printf("[event] %s -> %s\n", channel, payload);
}

evo_subscription_t *sub = evo_subscribe(
    "127.0.0.1", 9967, "admin", "admin",
    "agent_events", on_event, NULL);
if (!sub) {
    fprintf(stderr, "subscribe failed: %s\n", evo_strerror(NULL));
    return 1;
}

/* …main loop runs as long as the agent is alive… */

evo_unsubscribe(sub);   /* deterministic: stops the worker, joins, frees */

To produce events, another session runs NOTIFY agent_events, 'hello world';. You can do this with evosql-cli.

Formatting a vector

evo_vector_format() turns a float[] into the [v1,v2,...] literal. EvoSQL accepts this literal as a VECTOR(N) value. You then pass it as the optional embedding argument to evo_memory_put().

float emb[] = { 0.1f, 0.2f, 0.3f };
char lit[256];
evo_vector_format(emb, 3, lit, sizeof(lit));   /* lit = "[0.100000,0.200000,0.300000]" */
evo_memory_put(c, "demo", "users", "alice", "{\"role\":\"pm\"}", lit);

Reference

The full signatures and documentation live in include/evosql_memory.h. Every evo_* call returns an evo_error_t code. Some calls return a row count or NULL instead, as noted below. The error codes are EVO_OK (0), EVO_ERR (-1), EVO_ERR_CONNECT (-2), EVO_ERR_AUTH (-3), EVO_ERR_PROTOCOL (-4), EVO_ERR_SQL (-5), EVO_ERR_BUFFER (-6), and EVO_ERR_NOT_FOUND (-7).

Lifecycle

evo_conn_t *evo_connect(const char *host, int port, const char *user, const char *password)

Opens a connection and runs the EVO handshake and auth. It returns NULL on failure. Call evo_strerror(NULL) to get the reason. The returned handle is opaque. Free it with evo_close().

void evo_close(evo_conn_t *conn)

Sends a best-effort QUIT, closes the socket, and frees the handle. It is safe to call with NULL.

Generic SQL

int evo_exec(evo_conn_t *conn, const char *sql)

Runs a non-SELECT statement. It returns the number of affected rows, read from the server's TAG line. It returns 0 when there are none, or -1 on error.

int evo_query(evo_conn_t *conn, const char *sql, char *out, int max_rows, int max_cols, int col_buf_size, int *cols_out)

A helper for SELECT. The caller supplies one flat char buffer of max_rows × max_cols × col_buf_size bytes. The field at row r, column c lives at out + (r * max_cols + c) * col_buf_size. NULL fields are stored as the empty string. It returns the row count, or 0 for a non-SELECT, or -1 on error. cols_out is optional. It receives the column count.

Memory store

int evo_memory_put(evo_conn_t *conn, const char *store, const char *ns, const char *key, const char *value_json, const char *embedding_or_null)

Emits MEMORY PUT INTO <store> VALUES (ns, key, json [, embedding]). Pass NULL for embedding_or_null to skip the embedding. Otherwise pass a [0.1,0.2,...] literal. See evo_vector_format for how to build one. The library escapes single quotes in ns, key, and value_json.

int evo_memory_get(evo_conn_t *conn, const char *store, const char *ns, const char *key, char *out_value, int out_size)

Runs MEMORY GET FROM <store> WHERE NS = ? AND KEY = ?. It writes the value JSON into out_value. The value is NUL-terminated. It returns EVO_OK, or EVO_ERR_NOT_FOUND when no row matches.

int evo_memory_delete(evo_conn_t *conn, const char *store, const char *ns, const char *key)

Runs MEMORY DELETE FROM <store> WHERE NS = ? AND KEY = ?. It returns EVO_OK on success.

Checkpoint store

int evo_checkpoint_put(evo_conn_t *conn, const char *store, const char *thread_id, const char *checkpoint_ns, const char *checkpoint_id, const char *values_json, const char *metadata_json)

Emits CHECKPOINT PUT INTO <store> VALUES (...). The backing table has 7 fields. The SDK exposes the 5 fields the agent layer cares about. It supplies empty defaults for parent linkage. checkpoint_ns and metadata_json may be NULL. They default to empty and {}.

int evo_checkpoint_get_latest(evo_conn_t *conn, const char *store, const char *thread_id, char *out_value, int out_size)

Runs CHECKPOINT GET FROM <store> THREAD ?. It writes the latest checkpoint's values JSON into out_value. It returns EVO_OK or EVO_ERR_NOT_FOUND.

Vectors

int evo_vector_format(const float *vec, int n, char *out, int out_size)

Formats n floats as a [v1,v2,...] literal. It uses six decimal places. It returns the number of bytes written, not counting the NUL, or -1 on overflow.

Streaming

evo_subscription_t *evo_subscribe(const char *host, int port, const char *user, const char *password, const char *channel, evo_event_callback_t callback, void *ctx)

Opens its own authenticated connection and runs LISTEN <channel>. It sends every NOTIFY to callback. The LISTEN response is read synchronously, so startup errors appear before the worker thread starts. It returns NULL when the connect or listen fails. For NOTIFY events, lsn is always 0.

evo_subscription_t *evo_cdc_subscribe(const char *host, int cdc_port, evo_event_callback_t callback, void *ctx, uint64_t from_lsn)

Connects to the CDC JSON-lines stream and emits one callback per WAL change. The default port is 9970. It is a plain TCP port with no EVO handshake or auth. The callback's channel is the table name taken from the JSON. payload is the full JSON line. lsn carries the xid when it is present. from_lsn is reserved for v2. Pass 0.

int evo_ack_up_to(evo_subscription_t *sub, uint64_t lsn)

Records the highest acked LSN. In v1 this is stored locally only. Server-side acknowledgement is a v2 item. It lands once durable CDC subscription queues are available. NOTIFY subscriptions ignore the call. It always returns EVO_OK.

void evo_unsubscribe(evo_subscription_t *sub)

Tears down the subscription in a deterministic way. It flags the worker and calls shutdown() so the blocked recv() returns. It then joins the thread and frees all resources. The callback will not fire after this returns. It is safe to call with NULL.

typedef void (*evo_event_callback_t)(const char *channel, const char *payload, uint64_t lsn, void *ctx)

Your callback runs on the subscription's worker thread. Treat the arguments as borrowed pointers. Copy what you need, then return.

Error inspection

const char *evo_strerror(const evo_conn_t *conn)

Returns the last error message on the calling thread. The message is thread-local. It is never NULL. It returns "(no error)" when there is no error. The conn argument is accepted but ignored. Pass NULL after a failed evo_connect().

const char *evo_sqlstate(const evo_conn_t *conn)

Returns the SQLSTATE of the last error. It returns the empty string when there was none.

Notes & limitations

  • Threading. A single evo_conn_t * is not thread-safe. Open one connection per thread. The last-error string is in thread-local storage. Errors do not leak across threads. Subscriptions own their own socket and worker. A single subscription never re-enters its own callback because it holds an internal mutex. Two different subscriptions can run their callbacks at the same time.
  • Plaintext only (v1). There is no TLS client, no SCRAM, and no extended-query or batch protocol. The SDK declines STARTTLS and uses the legacy cleartext AUTH fallback.
  • CDC ack is local-only in v1. evo_ack_up_to() records the LSN on the client side. Durable server-side acknowledgement is a v2 item.
  • Fixed-size buffers. The typed wrappers stage SQL and results in fixed stack buffers. Value JSON for puts is up to about 16 KB. Oversized arguments return EVO_ERR_BUFFER. For very large or unusual statements, use evo_query or evo_exec directly.
  • SQL escaping. The wrappers double single quotes in ns, key, and JSON values. Store and identifier names are assumed to be already safe. Do not feed untrusted store names into the typed helpers.
