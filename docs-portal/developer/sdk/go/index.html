Go SDK

A cgo binding over libevosql-memory. It gives Go programs a clean Conn type, typed *Error returns, and the EvoSQL agent-memory functions. It talks to the server over the EVO text protocol.

What it does

The Go SDK (go-evosql-memory) is a thin wrapper around the C SDK libevosql-memory. It is not a pure-Go driver. The cgo layer links the C archive. It forwards each call to the matching evo_* function in evosql_memory.h. On the wire it speaks the EVO text protocol on default port 9967. This is the same protocol used by the CLI client and the MCP server.

The wrapper adds these improvements over raw cgo.

  • A Conn struct with a finalizer-backed Close(). A forgotten connection is still released by the GC.
  • A typed *Error. It carries a Kind, a Message, and a SQLState. The message comes from the C SDK thread-local last-error storage.
  • Helpers for the MEMORY STORE and CHECKPOINT STORE agent-memory functions. It also gives you generic Exec and Query calls for everything else.
  • A FormatVector helper. It renders a []float32 as the [v1,v2,…] literal that EvoSQL accepts as a VECTOR(N) value.
info
Scope note. This binding wraps the connection, generic SQL, MEMORY STORE, CHECKPOINT STORE, and vector-format parts of the C SDK. The C header also has a LISTEN/NOTIFY and CDC subscription API. These are evo_subscribe, evo_cdc_subscribe, and evo_unsubscribe. The Go package does not bind those calls yet. Use the C SDK or another SDK for streaming.

Install

The module path is shown below.

github.com/alptekin/evolutiondb/client/go-evosql-memory

The package needs cgo. The C archive libevosql-memory.a must be present at build time. Inside the monorepo the cgo directives use ${SRCDIR}. The relative path resolves on its own. Build the C SDK first.

# build the C SDK archive first
make -C ../libevosql-memory

# then build / run from the Go package directory
cd client/go-evosql-memory
go build ./...
go run ./examples/put_get

The in-tree cgo directives are shown below.

#cgo CFLAGS:  -I${SRCDIR}/../libevosql-memory/include
#cgo LDFLAGS: -L${SRCDIR}/../libevosql-memory/build -levosql-memory -lpthread

For a system-wide install, point cgo at the installed headers and library. Use the standard environment variables.

CGO_CFLAGS="-I/usr/local/include" \
CGO_LDFLAGS="-L/usr/local/lib -levosql-memory -lpthread" \
go build ./...
download
Pre-built libevosql-memory binaries for Linux, macOS, and Windows are attached to every GitHub Release. Unpack one and point CGO_LDFLAGS=-L… -levosql-memory at it. This skips the make -C ../libevosql-memory step.

Configure (connection & auth)

There are no environment variables for connecting. Pass host, port, user, and password straight to Connect. The default login on a fresh EvoSQL server is admin / admin. The EVO text protocol listens on port 9967.

c, err := evosql.Connect("127.0.0.1", 9967, "admin", "admin")
if err != nil { /* handle */ }
defer c.Close()

The only build-time setup is the cgo CGO_CFLAGS and CGO_LDFLAGS pair shown under Install. It tells the build where to find the C headers and archive.

Usage

The example below is the package's own smoke test. It lives in examples/put_get/main.go. It creates a MEMORY STORE, puts a JSON value, reads it back, then drops the store.

package main

import (
	"fmt"
	"log"

	evosql "github.com/alptekin/evolutiondb/client/go-evosql-memory"
)

func main() {
	c, err := evosql.Connect("127.0.0.1", 9967, "admin", "admin")
	if err != nil {
		log.Fatalf("connect: %v", err)
	}
	defer c.Close()

	c.Exec("DROP MEMORY STORE IF EXISTS go_demo")
	c.Exec("CREATE MEMORY STORE go_demo")

	if err := c.MemoryPut("go_demo", "users", "alice",
		`{"role":"pm","team":"infra"}`, nil); err != nil {
		log.Fatalf("put: %v", err)
	}

	val, err := c.MemoryGet("go_demo", "users", "alice")
	if err != nil {
		log.Fatalf("get: %v", err)
	}
	fmt.Println("alice ->", val)

	c.Exec("DROP MEMORY STORE go_demo")
}

Generic SQL

For anything the typed helpers do not cover, use Exec and Query. Exec runs DML and DDL and returns the affected-row count. Query runs a SELECT and returns rows of strings. The EVO wire format is text, so NULL columns come back as "".

n, _ := c.Exec("CREATE TABLE t (id INT, name VARCHAR(50))")

rows, err := c.Query("SELECT id, name FROM t")
if err != nil { /* ... */ }
for _, row := range rows {
	fmt.Println(row[0], row[1])
}
warning
Query uses fixed buffers. It holds at most 256 rows, 32 columns, and 4096 bytes per cell. Result sets larger than these limits are cut to fit the buffer. For wide or large results, narrow the query or page it yourself.

Vectors / embeddings

MemoryPut takes an optional []float32 embedding. Pass nil for no embedding. A non-empty slice is rendered with FormatVector into the [v1,v2,…] literal before it is sent.

emb := []float32{0.12, 0.34, 0.56}
c.MemoryPut("go_demo", "docs", "d1", `{"text":"hello"}`, emb)

lit := evosql.FormatVector(emb) // "[0.120000,0.340000,0.560000]"

Reference

All methods hang off *Conn unless noted. Errors are returned as error. Under the hood they are always *Error. See Errors.

func Connect(host string, port int, user, password string) (*Conn, error)

Opens a connection to an EvoSQL server over the EVO text protocol. Returns nil and an *Error on failure. A finalizer is registered, so the connection is released if you forget to Close it. You should still call Close yourself.

func (c *Conn) Close() error

Releases the connection. It is safe to call more than once. It is also safe when the finalizer calls it. Always returns nil.

func (c *Conn) Exec(sql string) (int, error)

Runs a DML or DDL statement and returns the affected-row count. A negative C return code is turned into a typed *Error.

func (c *Conn) Query(sql string) ([][]string, error)

Runs a SELECT and returns rows where each cell is a string. NULL columns come back as the empty string. The fixed 256×32×4096 buffer limits noted above still apply.

func (c *Conn) MemoryPut(store, namespace, key, valueJSON string, embedding []float32) error

Writes a row into a MEMORY STORE under (namespace, key). embedding may be nil. A non-empty slice is formatted to a vector literal first.

func (c *Conn) MemoryGet(store, namespace, key string) (string, error)

Returns the value JSON for (store, namespace, key). When the row is missing it returns an *Error with Kind == ErrNotFound. The internal read buffer is 32 KB.

func (c *Conn) MemoryDelete(store, namespace, key string) error

Deletes one row from a MEMORY STORE.

func (c *Conn) CheckpointPut(store, threadID, ns, checkpointID, valuesJSON, metadataJSON string) error

Writes a checkpoint snapshot into a CHECKPOINT STORE for the given thread.

func (c *Conn) CheckpointGetLatest(store, threadID string) (string, error)

Returns the latest checkpoint values JSON for the thread. If there is none, it returns an *Error with Kind == ErrNotFound. The internal read buffer is 64 KB.

func FormatVector(vec []float32) string

Package-level helper. Renders a []float32 as the [v1,v2,…] literal that EvoSQL accepts as a VECTOR(N) value. Each component is formatted with %.6f. Returns the empty string for an empty or nil slice.

Errors

Every call that can fail returns a *Error. It is assignable to error. It carries three fields.

  • Kind ErrorKind. It is one of ErrGeneral, ErrConnect, ErrAuth, ErrProtocol, ErrSQL, ErrBuffer, or ErrNotFound. It is mapped from the C SDK return codes -2 through -7.
  • Message string. It is the human-readable message from the C SDK thread-local last-error storage.
  • SQLState string. It is the SQLSTATE of the last error, or empty when there is none.

When a SQLSTATE is present, Error() formats as evosql sql <state>: <message>. Otherwise it formats as evosql: <message>. Use a type assertion or errors.As to inspect Kind. The example below treats a missing row as a soft miss.

val, err := c.MemoryGet("go_demo", "users", "bob")
var e *evosql.Error
if errors.As(err, &e) && e.Kind == evosql.ErrNotFound {
	// row absent - handle gracefully
}

Notes & limitations

  • cgo required. This is not a pure-Go driver. You must build or download libevosql-memory and have a C toolchain ready. Cross-compiling needs the matching C archive for the target platform.
  • A *Conn is not goroutine-safe. Open one connection per goroutine. The C SDK last-error storage is thread-local. Two goroutines on two connections will not see each other's errors through Error.Message.
  • Buffer limits. Query is capped at 256 rows by 32 columns by 4096 bytes per cell. MemoryGet reads up to 32 KB. CheckpointGetLatest reads up to 64 KB. Larger payloads are cut off.
  • No streaming binding. The C SDK underneath supports LISTEN/NOTIFY and CDC subscriptions. The Go package does not wrap them.
  • Go version. The module declares go 1.21.

See also the MCP server for agent integrations. The project wiki has the EVO protocol and agent-memory store reference.
