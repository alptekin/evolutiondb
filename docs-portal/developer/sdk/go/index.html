Go SDK

A cgo binding over libevosql-memory that gives Go programs an idiomatic Conn type, typed *Error returns, and the EvoSQL agent-memory primitives, talking to the server over the EVO text protocol.

What it does

The Go SDK (go-evosql-memory) is a thin, idiomatic wrapper around the C SDK libevosql-memory. It is not a pure-Go driver: the cgo layer links the C archive and forwards each call to the corresponding evo_* function in evosql_memory.h. On the wire it speaks the EVO text protocol (default port 9967), the same protocol used by the CLI client and the MCP server.

What you get over raw cgo:

  • A Conn struct with a finalizer-backed Close() so a forgotten connection is still released by the GC.
  • A typed *Error carrying a Kind, a Message (from the C SDK's thread-local last-error storage), and a SQLState.
  • Helpers for the MEMORY STORE and CHECKPOINT STORE agent-memory primitives, plus generic Exec/Query escape hatches.
  • A FormatVector helper that renders a []float32 as the [v1,v2,…] literal EvoSQL accepts as a VECTOR(N) value.
info
Scope note: this binding wraps the connection, generic SQL, MEMORY STORE, CHECKPOINT STORE, and vector-format surfaces of the C SDK. The C header also exposes a LISTEN/NOTIFY and CDC subscription API (evo_subscribe, evo_cdc_subscribe, evo_unsubscribe), but the Go package does not bind those calls yet, use the C or another SDK for streaming.

Install

The module path is:

github.com/alptekin/evolutiondb/client/go-evosql-memory

The package requires cgo and the C archive libevosql-memory.a to be present at build time. Inside the monorepo the cgo directives use ${SRCDIR} so the relative path resolves out of the box, just build the C SDK first:

# build the C SDK archive first
make -C ../libevosql-memory

# then build / run from the Go package directory
cd client/go-evosql-memory
go build ./...
go run ./examples/put_get

The in-tree cgo directives are:

#cgo CFLAGS:  -I${SRCDIR}/../libevosql-memory/include
#cgo LDFLAGS: -L${SRCDIR}/../libevosql-memory/build -levosql-memory -lpthread

For a system-wide install, point cgo at the installed headers and library with the standard environment variables:

CGO_CFLAGS="-I/usr/local/include" \
CGO_LDFLAGS="-L/usr/local/lib -levosql-memory -lpthread" \
go build ./...
download
Pre-built libevosql-memory binaries for Linux / macOS / Windows are attached to every GitHub Release. Unpack one and point CGO_LDFLAGS=-L… -levosql-memory at it to skip the make -C ../libevosql-memory step.

Configure (connection & auth)

There are no environment variables for connecting: host, port, user, and password are passed directly to Connect. The default credentials on a fresh EvoSQL server are admin / admin, and the EVO text protocol listens on port 9967.

c, err := evosql.Connect("127.0.0.1", 9967, "admin", "admin")
if err != nil { /* handle */ }
defer c.Close()

The only build-time configuration is the cgo CGO_CFLAGS / CGO_LDFLAGS pair shown under Install, used to locate the C headers and archive.

Usage

The example below is the package's own smoke test (examples/put_get/main.go): create a MEMORY STORE, put a JSON value, read it back, then drop the store.

package main

import (
	"fmt"
	"log"

	evosql "github.com/alptekin/evolutiondb/client/go-evosql-memory"
)

func main() {
	c, err := evosql.Connect("127.0.0.1", 9967, "admin", "admin")
	if err != nil {
		log.Fatalf("connect: %v", err)
	}
	defer c.Close()

	c.Exec("DROP MEMORY STORE IF EXISTS go_demo")
	c.Exec("CREATE MEMORY STORE go_demo")

	if err := c.MemoryPut("go_demo", "users", "alice",
		`{"role":"pm","team":"infra"}`, nil); err != nil {
		log.Fatalf("put: %v", err)
	}

	val, err := c.MemoryGet("go_demo", "users", "alice")
	if err != nil {
		log.Fatalf("get: %v", err)
	}
	fmt.Println("alice ->", val)

	c.Exec("DROP MEMORY STORE go_demo")
}

Generic SQL

For anything the typed helpers don't cover, use Exec (DML/DDL, returns the affected-row count) and Query (SELECT, returns rows of strings: the EVO wire format is text, so NULL columns come back as "").

n, _ := c.Exec("CREATE TABLE t (id INT, name VARCHAR(50))")

rows, err := c.Query("SELECT id, name FROM t")
if err != nil { /* ... */ }
for _, row := range rows {
	fmt.Println(row[0], row[1])
}
warning
Query uses fixed buffers: at most 256 rows, 32 columns, and 4096 bytes per cell. Result sets larger than these limits are truncated to the buffer. For wide or large results, narrow the query or page it yourself.

Vectors / embeddings

MemoryPut takes an optional []float32 embedding. Pass nil for no embedding; a non-empty slice is rendered with FormatVector into the [v1,v2,…] literal before being sent.

emb := []float32{0.12, 0.34, 0.56}
c.MemoryPut("go_demo", "docs", "d1", `{"text":"hello"}`, emb)

lit := evosql.FormatVector(emb) // "[0.120000,0.340000,0.560000]"

Reference

All methods hang off *Conn unless noted. Errors are returned as error but concretely are *Error (see Errors).

func Connect(host string, port int, user, password string) (*Conn, error)

Opens a connection to an EvoSQL server over the EVO text protocol. Returns nil and an *Error on failure. A finalizer is registered so the connection is released if you forget to Close it; the caller is still responsible for calling Close explicitly.

func (c *Conn) Close() error

Releases the connection. Idempotent, safe to call more than once and safe when invoked by the finalizer. Always returns nil.

func (c *Conn) Exec(sql string) (int, error)

Runs a DML/DDL statement and returns the affected-row count. A negative C return code is translated into a typed *Error.

func (c *Conn) Query(sql string) ([][]string, error)

Runs a SELECT and returns rows where each cell is a string. NULL columns come back as the empty string. Subject to the fixed 256×32×4096 buffer limits noted above.

func (c *Conn) MemoryPut(store, namespace, key, valueJSON string, embedding []float32) error

Writes a row into a MEMORY STORE under (namespace, key). embedding may be nil; a non-empty slice is formatted to a vector literal first.

func (c *Conn) MemoryGet(store, namespace, key string) (string, error)

Returns the value JSON for (store, namespace, key). When the row is missing it returns an *Error with Kind == ErrNotFound. The internal read buffer is 32 KB.

func (c *Conn) MemoryDelete(store, namespace, key string) error

Deletes one row from a MEMORY STORE.

func (c *Conn) CheckpointPut(store, threadID, ns, checkpointID, valuesJSON, metadataJSON string) error

Writes a checkpoint snapshot into a CHECKPOINT STORE for the given thread.

func (c *Conn) CheckpointGetLatest(store, threadID string) (string, error)

Returns the latest checkpoint values JSON for the thread, or an *Error with Kind == ErrNotFound if there is none. The internal read buffer is 64 KB.

func FormatVector(vec []float32) string

Package-level helper. Renders a []float32 as the [v1,v2,…] literal EvoSQL accepts as a VECTOR(N) value, formatting each component with %.6f. Returns the empty string for an empty/nil slice.

Errors

Every fallible call returns a *Error (assignable to error). It carries:

  • Kind ErrorKind, one of ErrGeneral, ErrConnect, ErrAuth, ErrProtocol, ErrSQL, ErrBuffer, ErrNotFound, mapped from the C SDK return codes (-2…-7).
  • Message string, the human-readable message from the C SDK's thread-local last-error storage.
  • SQLState string: the SQLSTATE of the last error, or empty when there is none.

Error() formats as evosql sql <state>: <message> when a SQLSTATE is present, otherwise evosql: <message>. Use a type assertion or errors.As to inspect Kind: for example, treating a missing row as a soft miss:

val, err := c.MemoryGet("go_demo", "users", "bob")
var e *evosql.Error
if errors.As(err, &e) && e.Kind == evosql.ErrNotFound {
	// row absent - handle gracefully
}

Notes & limitations

  • cgo required. This is not a pure-Go driver. You must build (or download) libevosql-memory and have a C toolchain available; cross-compiling needs the matching C archive for the target platform.
  • A *Conn is not goroutine-safe. Open one connection per goroutine. The C SDK's last-error storage is thread-local, so two goroutines on two connections won't see each other's errors via Error.Message.
  • Buffer limits. Query is capped at 256 rows × 32 columns × 4096 bytes/cell; MemoryGet reads up to 32 KB; CheckpointGetLatest up to 64 KB. Larger payloads are truncated.
  • No streaming binding. The underlying C SDK supports LISTEN/NOTIFY and CDC subscriptions, but the Go package does not wrap them.
  • Go version. The module declares go 1.21.

See also: MCP server for agent integrations, and the project wiki for the EVO protocol and agent-memory store reference.
