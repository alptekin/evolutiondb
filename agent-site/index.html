How It Works

EvolutionAgent follows four simple steps. Each step runs automatically. You stay in control at every point.

01
link

Connect Your Tools

You connect your accounts. Gmail, GitHub, Slack, Teams, Notion, Calendar, and more. The agent reads from these sources on a schedule you set.

02
sync

Memory Is Built

Every event, message, and update is stored in EvolutionDB as structured SQL. The memory grows with you. Nothing is lost between runs.

03
auto_fix_high

Agent Takes Action

The agent reads your memory, understands the context, and prepares a response or task. It writes a draft. It does not act on its own yet.

04
verified_user

You Approve

You read the draft. You approve it, edit it, or reject it. Only approved actions are sent. The agent never acts without your permission.

Daily Brief

Your Morning Brief

EvolutionAgent can email you a short brief each morning. It reads from the memory it already keeps, then sums up your day in three parts.

history

Yesterday's Recap

Activity counts by source, like 7 emails and 3 Slack messages, plus the meetings you had that day.

today

Today's Schedule

Your calendar meetings for today, each with its start time and title, so you know what is coming.

pending_actions

What's Pending

Open loops in one place: who is waiting on you and at what priority, what you are waiting on, and the promises you made.

verified_user

The brief goes to you and only you, so it can send itself once you opt in. Every other outbound reply still waits for your explicit approval.

Built for Trust

EvolutionAgent is designed around one principle. The agent assists. The human decides.

database

SQL Memory

All memories are stored in EvolutionDB as real SQL tables. You can query, export, or delete them at any time. Your data is always yours.

verified_user

Human Approval

Every message, reply, or task goes into an outbox first. You read it and approve it. Nothing reaches the outside world without your sign-off.

extension

MCP Server

EvolutionAgent ships an MCP server. You can connect it to any Claude-powered tool or IDE. Your memory becomes available wherever Claude runs.

Open Source. Written in Python and C.

The agent loop is written in Python. The memory engine is EvolutionDB, written in C. The two components are clean and separate. You can inspect, fork, or extend either one.

The MCP server follows the Model Context Protocol spec. Any MCP-compatible client can connect to it. You are not locked in.
