USE

USE picks the active database for the current session.

Synopsis

USE [ DATABASE ] database_name

Description

USE changes which database your next statements run against. EvolutionDB finds every table through the active database and schema pair. When you run USE, all unqualified object lookups point to the named database.

The target database must already exist in the system catalog. If it does not exist, the command fails with database "database_name" does not exist. This is the SQLSTATE for an undefined database. The active database stays the same.

On a successful switch, the active schema also resets to default. Any earlier schema choice is dropped by the switch. For example, a choice made with SET SCHEMA is lost. If you want a non-default schema in the new database, you must choose it again after the switch.

The optional DATABASE keyword is only for looks. The grammar accepts both USE mydb and USE DATABASE mydb. They behave the same way.

Note: USE changes session state, not stored schema. It is not transactional. A failed statement does not roll it back. It takes effect right away for the connection that runs it. It stays in effect until the next USE. It also ends when the connection closes.

Parameters

DATABASE
Optional noise keyword. It is there only to read well and to feel familiar in the MySQL style. It does not change behavior. Both USE name and USE DATABASE name do the same thing.
database_name
The name of the database to make active. It must be a database that already exists in the system catalog. The lookup runs against the catalog. A missing database gives an error. It does not create one. After a successful switch, the active schema resets to default.

Examples

This example switches to a database named sales using the short form.

USE sales;

This example uses the optional DATABASE keyword to do the same switch.

USE DATABASE sales;

This example creates a database and then starts working inside it. The switch resets the schema to default, so the CREATE TABLE below goes into sales.default.

CREATE DATABASE sales;
USE sales;
CREATE TABLE orders (
    id      INT PRIMARY KEY,
    total   DECIMAL(12, 2)
);

This example shows how to choose a non-default schema after switching databases. USE resets the schema first, so the explicit SET SCHEMA must come after it.

USE DATABASE sales;   -- active schema is now "default"
SET SCHEMA archive;  -- active schema is now "archive"

Compatibility

USE is not part of the SQL standard. EvolutionDB follows the MySQL way. It is a session-level command that picks the current database. EvolutionDB also accepts the optional DATABASE noise keyword.

PostgreSQL has no USE statement. In PostgreSQL a connection is tied to one database for its whole life. To switch databases you must open a new connection. For example, the psql meta-command \c does this. EvolutionDB instead lets one connection move between databases while it runs.

There is one more difference from PostgreSQL. The EvolutionDB default schema is named default, not public. A successful USE always resets the active schema to default.
