CREATE DATABASE

Create a new, empty database and add it to the system catalog. EvolutionDB also makes a schema named default inside it.

Synopsis

CREATE DATABASE [ IF NOT EXISTS ] database_name

Description

CREATE DATABASE makes a new database. A database is the top container in the EvolutionDB catalog. It holds schemas. Schemas hold tables and other objects. The statement saves the new database in the system catalog. The catalog gives it a database id.

When it creates the database, EvolutionDB also makes a schema named default inside it. So the database is ready for new objects right away. You do not need a separate CREATE SCHEMA step.

Creating a database does not switch your session to it. Your current database stays the same. Use USE to make the new database current. Do this before you create objects in it.

What happens if a database with the same name already exists? The statement fails with a "database already exists" error. The SQLSTATE is 42E04. You can avoid this with IF NOT EXISTS. Then the existing database is left as is, and the statement succeeds with no error. The statement can also fail for another reason. If the catalog cannot allocate the database, you get an I/O error. The SQLSTATE is 58030.

A new instance starts with a default database named evosql. That database also has a default schema inside it. So at least one database always exists before you run any CREATE DATABASE.

Parameters

IF NOT EXISTS
Hides the "database already exists" error. If the catalog already has a database with this name, no new database is made. The statement still succeeds, and it does not change the existing database. Without this clause, creating a database that already exists is an error.
database_name
The name of the database to create. It must be unique among the databases that already exist. The name lookup against the catalog is case-sensitive.

Examples

This example creates a database.

CREATE DATABASE shop;

Create a database only if it does not already exist. You can run this many times and get the same result.

CREATE DATABASE IF NOT EXISTS analytics;

Create a database. Then switch to it and start defining objects in its default schema.

CREATE DATABASE reporting;
USE reporting;
CREATE TABLE events (
    id   BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(120) NOT NULL
);

Compatibility

The SQL standard does not define CREATE DATABASE. It is a PostgreSQL and MySQL extension. EvolutionDB keeps its version of the command small on purpose.

  • Versus PostgreSQL: PostgreSQL's CREATE DATABASE takes a long list of options. These include OWNER, TEMPLATE, ENCODING, LOCALE, LC_COLLATE, LC_CTYPE, TABLESPACE, CONNECTION LIMIT, and IS_TEMPLATE. EvolutionDB supports none of these. The grammar does not accept them. The only optional part is the IF NOT EXISTS guard.
  • Versus MySQL: MySQL allows CREATE {DATABASE | SCHEMA} with CHARACTER SET and COLLATE options. EvolutionDB does not accept those options on CREATE DATABASE. In EvolutionDB, CREATE DATABASE and CREATE SCHEMA are not the same thing. They are separate commands at different levels. A schema lives inside the current database.
  • EvolutionDB behavior: Creating a database also creates a schema named default inside it. The new database does not become your current database. You must run USE for that.
