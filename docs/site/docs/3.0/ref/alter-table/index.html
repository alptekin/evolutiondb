ALTER TABLE

Change the definition of a table that already exists. You can add, drop, rename, or retype columns. You can add, drop, rename, enable, disable, or validate constraints. You can turn row-level security on or off. You can also set or drop a TTL column.

Synopsis

ALTER TABLE table_name action

where action is one of:

-- columns
  ADD COLUMN column_name data_type [ column_attribute ... ] [ FIRST | AFTER existing_column ]
  DROP COLUMN column_name
  DROP column_name
  RENAME COLUMN old_column TO new_column
  RENAME old_column TO new_column
  MODIFY COLUMN column_name data_type [ column_attribute ... ] [ FIRST | AFTER existing_column ]
  MODIFY column_name data_type [ column_attribute ... ] [ FIRST | AFTER existing_column ]
  CHANGE COLUMN old_column new_column data_type [ column_attribute ... ] [ FIRST | AFTER existing_column ]
  CHANGE old_column new_column data_type [ column_attribute ... ] [ FIRST | AFTER existing_column ]

-- constraints
  ADD CONSTRAINT constraint_name CHECK ( expression ) [ NOT VALIDATE ]
  ADD CONSTRAINT constraint_name UNIQUE ( column [, ...] )
  ADD CONSTRAINT constraint_name PRIMARY KEY ( column [, ...] )
  ADD CONSTRAINT constraint_name FOREIGN KEY ( column [, ...] )
      REFERENCES ref_table ( ref_column [, ...] ) [ referential_action ... ] [ NOT VALIDATE ]
  DROP CONSTRAINT constraint_name
  RENAME CONSTRAINT old_name TO new_name
  ENABLE CONSTRAINT constraint_name
  DISABLE CONSTRAINT constraint_name
  VALIDATE CONSTRAINT constraint_name

-- row-level security
  ENABLE ROW LEVEL SECURITY
  DISABLE ROW LEVEL SECURITY

-- time-to-live
  SET TTL_COLUMN = 'column_name'
  DROP TTL

and referential_action is one of:

  ON DELETE { CASCADE | SET NULL | SET DEFAULT | RESTRICT | NO ACTION }
  ON UPDATE { CASCADE | SET NULL | SET DEFAULT | RESTRICT | NO ACTION }
  MATCH { FULL | PARTIAL | SIMPLE }
  [ NOT ] DEFERRABLE [ INITIALLY { DEFERRED | IMMEDIATE } ]

Description

ALTER TABLE changes the definition of a table that already exists. Each statement does exactly one action. EvolutionDB does not accept a list of actions separated by commas in one ALTER TABLE command. Write a separate statement for each change.

If the named table does not exist, the command fails. It returns relation "table_name" does not exist with SQLSTATE 42P01. All actions work on the system catalog. The catalog stores tables, columns, and constraints in B+ trees.

The column actions use two different spellings. ADD COLUMN, RENAME COLUMN ... TO, and DROP COLUMN use the ANSI and PostgreSQL spelling. MODIFY and CHANGE use the MySQL spelling. The COLUMN keyword is optional for DROP, RENAME, MODIFY, and CHANGE. MODIFY changes the type of a column in place. CHANGE renames and retypes a column in one statement.

Adding, dropping, and reordering columns can rewrite the existing rows. ADD COLUMN with a FIRST or AFTER position moves the column in the catalog. It then rebuilds every tuple and fills the new column with NULL. MODIFY and CHANGE with a position clause also rewrite all tuples in the new column order. DROP COLUMN marks the column as dropped in the catalog. It does not delete the data right away.

Adding a CHECK, PRIMARY KEY, or UNIQUE constraint checks the existing rows first. The command stops if any row breaks the new constraint. You can skip this check with NOT VALIDATE, which works for CHECK and FOREIGN KEY only. NOT VALIDATE records the constraint as not-validated and skips the row scan. You can run VALIDATE CONSTRAINT later to do the check that was skipped.

On success, column, TTL, and RLS actions report ALTER TABLE. Constraint actions report command(s) completed successfully!...

Parameters

table_name
The name of the table you want to change. The name is looked up in the current database and schema.
ADD COLUMN column_name data_type [ column_attribute.. ]
Adds a new column to the table. The column goes at the end unless you give a position clause. The command fails with 42701 if a column with that name already exists. It fails with 54011 if the table already has the maximum number of columns. The attributes are the same as in CREATE TABLE. They include NOT NULL, NULL, DEFAULT, UNIQUE, AUTO_INCREMENT / IDENTITY, CHECK (...), COMMENT, and GENERATED ALWAYS AS (...) STORED|VIRTUAL. A DEFAULT can be a literal, CURRENT_TIMESTAMP, gen_random_uuid(), gen_random_uuid_v7(), snowflake_id(), or an expression. A column added this way is never made part of the primary key. The command fails with 23502 when the column is NOT NULL, has no DEFAULT, and the table already holds rows. This is because the existing rows would get a NULL.
FIRST | AFTER existing_column
An optional position clause for ADD, MODIFY, and CHANGE. FIRST puts the column at ordinal 0. AFTER existing_column puts it right after the named column. When you choose a position, every row is rewritten into the new physical column order.
DROP COLUMN column_name  /  DROP column_name
Marks a column as dropped. The command fails with 42703 if the column does not exist. It fails with 2BP01 in a few cases. These are when the column is part of the primary key, when it is the only active column left, when a secondary index uses it, or when a FOREIGN KEY or UNIQUE constraint uses it.
RENAME COLUMN old_column TO new_column  /  RENAME old_column TO new_column
Gives a column a new name. The COLUMN keyword is optional.
MODIFY [COLUMN] column_name data_type [ column_attribute.. ]
Changes the type or attributes of a column that already exists. The command fails with 42703 if the column does not exist. It fails with 2BP01 if the column is part of the primary key and its type would change. It fails with 42804 if the old type and the new type cannot be converted automatically. The new definition may add NOT NULL to a column that allowed NULL before. In that case the table is scanned. The command then fails with 23502 if any row holds a NULL in that column.
CHANGE [COLUMN] old_column new_column data_type [ column_attribute.. ]
A MySQL-style action that renames and retypes a column at the same time. It renames old_column to new_column when the names differ. It then applies the new type and attributes. It runs the same checks as MODIFY. A position clause works here too.
ADD CONSTRAINT constraint_name CHECK ( expression )
Adds a named CHECK constraint. Every existing row is tested against expression. If any row fails, the command stops with 23514 and reports the row number that failed. The command fails with 42710 if a constraint with the same name already exists.
ADD CONSTRAINT constraint_name UNIQUE ( column [..] )
Adds a named UNIQUE constraint over one or more columns. The command fails with 42710 if the constraint name is already used.
ADD CONSTRAINT constraint_name PRIMARY KEY ( column [..] )
Adds a primary key over the listed columns. The command fails with 42710 if the table already has a primary key. It fails with 42703 if a named column does not exist. It fails with the not-null violation code if any key column holds NULL values. It fails with the unique violation code if the existing rows hold duplicate key values.
ADD CONSTRAINT constraint_name FOREIGN KEY ( column [..] ) REFERENCES ref_table ( ref_column [..] ) [ referential_action.. ]
Adds a foreign key that references ref_table. The command fails with 42P01 if the referenced table does not exist. It fails with 42710 if the constraint name is already used.
referential_action
One or more clauses that you attach to a FOREIGN KEY. You can leave them out. ON DELETE and ON UPDATE each take CASCADE, SET NULL, SET DEFAULT, RESTRICT, or NO ACTION. MATCH takes FULL, PARTIAL, or SIMPLE. You can also mark the constraint [NOT] DEFERRABLE, and add INITIALLY DEFERRED or INITIALLY IMMEDIATE.
NOT VALIDATE
You can use this on ADD CONSTRAINT ... CHECK and ADD CONSTRAINT ... FOREIGN KEY. It records the constraint as not-validated and skips the scan of existing rows. The constraint then applies only to later writes. It stays this way until you run VALIDATE CONSTRAINT.
DROP CONSTRAINT constraint_name
Removes a constraint by name. The command fails with 42704 if the table has no constraint with that name.
RENAME CONSTRAINT old_name TO new_name
Gives a constraint a new name. The command fails with 42704 if old_name is not found. It fails with 42710 if new_name is already in use.
ENABLE CONSTRAINT constraint_name  /  DISABLE CONSTRAINT constraint_name
Turns the enabled flag of a constraint on or off. A disabled constraint is not enforced on later writes. The command fails with 42704 if the constraint does not exist.
VALIDATE CONSTRAINT constraint_name
Validates a constraint that was added with NOT VALIDATE. It checks the constraint against the existing rows. It then marks the constraint as validated.
ENABLE / DISABLE ROW LEVEL SECURITY
Sets or clears the row-level-security flag on the table. The keywords must be exactly ROW LEVEL SECURITY. Any other word after ROW is a syntax error.
SET TTL_COLUMN = 'column_name'
Sets a timestamp column as the time-to-live column for the table. The value is a string in single quotes. The command fails if the named column does not exist on the table.
DROP TTL
Clears the TTL column setting on the table.

Examples

Add a column that allows NULL. Then add a column with a default and place it after an existing column.

ALTER TABLE employees ADD COLUMN nickname VARCHAR(50);
ALTER TABLE employees ADD COLUMN created_at TIMESTAMP
    DEFAULT CURRENT_TIMESTAMP AFTER nickname;

Rename and retype columns. These examples show the PostgreSQL and MySQL spellings.

ALTER TABLE employees RENAME COLUMN nickname TO display_name;
ALTER TABLE employees MODIFY COLUMN display_name VARCHAR(100) NOT NULL;
ALTER TABLE employees CHANGE display_name full_name VARCHAR(120);

This example drops a column.

ALTER TABLE employees DROP COLUMN full_name;

These examples add different types of constraints.

ALTER TABLE employees ADD CONSTRAINT chk_age CHECK (age >= 18);
ALTER TABLE employees ADD CONSTRAINT uq_email UNIQUE (email);
ALTER TABLE employees ADD CONSTRAINT pk_emp PRIMARY KEY (id);
ALTER TABLE orders ADD CONSTRAINT fk_emp FOREIGN KEY (emp_id)
    REFERENCES employees (id) ON DELETE CASCADE;

Add a foreign key without checking the existing data. Then validate it later.

ALTER TABLE orders ADD CONSTRAINT fk_emp2 FOREIGN KEY (emp_id)
    REFERENCES employees (id) NOT VALIDATE;
ALTER TABLE orders VALIDATE CONSTRAINT fk_emp2;

This example renames a constraint, disables it, enables it again, then drops it.

ALTER TABLE employees RENAME CONSTRAINT chk_age TO chk_min_age;
ALTER TABLE employees DISABLE CONSTRAINT chk_min_age;
ALTER TABLE employees ENABLE CONSTRAINT chk_min_age;
ALTER TABLE employees DROP CONSTRAINT chk_min_age;

These examples turn row-level security on and off. They also set and drop a TTL column.

ALTER TABLE sessions ENABLE ROW LEVEL SECURITY;
ALTER TABLE sessions SET TTL_COLUMN = 'expires_at';
ALTER TABLE sessions DROP TTL;
ALTER TABLE sessions DISABLE ROW LEVEL SECURITY;

Compatibility

The SQL standard defines ALTER TABLE with the ADD, DROP, and ALTER COLUMN subforms. EvolutionDB mixes the PostgreSQL and MySQL spellings on purpose.

  • PostgreSQL-style forms that are supported. These are ADD COLUMN, DROP COLUMN, RENAME COLUMN ... TO, named ADD CONSTRAINT for CHECK / UNIQUE / PRIMARY KEY / FOREIGN KEY, DROP CONSTRAINT, RENAME CONSTRAINT, VALIDATE CONSTRAINT, the referential MATCH and DEFERRABLE clauses, and ENABLE / DISABLE ROW LEVEL SECURITY.
  • MySQL-style forms that are supported. These are MODIFY [COLUMN], CHANGE [COLUMN], the optional COLUMN keyword on DROP and RENAME, and the FIRST / AFTER column-position clauses.
  • Differences from PostgreSQL. PostgreSQL spells deferred validation NOT VALID. EvolutionDB uses NOT VALIDATE. PostgreSQL writes ALTER COLUMN ... TYPE and SET/DROP NOT NULL. EvolutionDB retypes a column through MODIFY and CHANGE instead. ENABLE / DISABLE CONSTRAINT is an EvolutionDB extension. PostgreSQL has no per-constraint enable flag. EvolutionDB does not accept several actions separated by commas in one statement.
  • EvolutionDB extensions. SET TTL_COLUMN = '...' and DROP TTL manage automatic row expiry. They have no equivalent in the SQL standard or in PostgreSQL.
  • Not supported. Some clauses are not part of the ALTER TABLE grammar. These are ALTER COLUMN ... SET DEFAULT, SET/DROP NOT NULL, OWNER TO, SET SCHEMA, SET TABLESPACE, INHERIT, and RENAME TO. RENAME TO renames a table and is a separate statement.
