SHOW

Show the value of a run-time configuration parameter. You can also list catalog objects such as databases, schemas, tables, users, grants, and stores.

Synopsis

SHOW DATABASES
SHOW SCHEMAS
SHOW TABLES
SHOW VIEWS
SHOW CREATE VIEW view_name
SHOW USERS
SHOW ROLES
SHOW GRANTS [ FOR user_name ]
SHOW JOBS
SHOW SUBSCRIPTIONS
SHOW MEMORY STORES
SHOW CHECKPOINT STORES
SHOW ENTITY STORES
SHOW GRAPH STORES
SHOW DOCUMENT STORES
SHOW MESSAGE LOGS
SHOW EVO_SESSIONS
SHOW EVO_PROCESSES
SHOW REPLICATION STATUS
SHOW REPLICATION LAG
SHOW NODES
SHOW TABLE PLACEMENT
SHOW SHARDS table_name
SHOW configuration_parameter

Description

SHOW reports the value of a run-time configuration parameter. It can also return a result set that lists catalog objects. Most EvolutionDB statements are defined in the Bison grammar. SHOW is not defined there. The grammar file is evolution/parser/evoparser.y. Instead, the dual-protocol adaptor catches any statement whose first keyword is SHOW. This happens in adaptor/catalog.c. The adaptor handles the statement before the SQL engine ever sees it.

Dispatch uses case-insensitive substring matching against the raw statement text. The first matching form wins. Three forms are handled by the distributed-command handler. These are SHOW NODES, SHOW TABLE PLACEMENT, and SHOW SHARDS. That handler runs ahead of the general SHOW handler. All forms return a result set tagged with the command tag SHOW. The object-listing forms behave like a SELECT. Each one defines its own columns.

Matching is substring-based and order-sensitive. The keyword you use must match first in the dispatch chain. For example, a statement that contains the word tables is treated as SHOW TABLES. A SHOW statement that matches none of the listed forms falls through to the run-time parameter path. That path returns a single setting column. The value is an empty string when the parameter is unknown.

Scope. The object listings use the session's current database and current schema. SHOW TABLES and SHOW VIEWS show only objects in the current schema. A temporary table is shown only to the session that owns it. SHOW DATABASES, SHOW USERS, SHOW ROLES, and the store, job, and subscription listings cover the whole cluster.

Parameters

DATABASES
Lists every database in the cluster. Returns one column, database.
SCHEMAS
Lists every schema in the current database. Returns one column, schema.
TABLES
Lists tables in the current schema of the current database. Returns one column, table. A temporary table appears only in the session that created it.
VIEWS
Lists views in the current schema. Returns view_name and view_sql. The view_sql column holds the stored query that defines the view.
CREATE VIEW view_name
Shows the rebuilt CREATE VIEW statement for the named view. Returns columns View and Create View. The definition is rebuilt as CREATE VIEW name AS <stored sql>. The view must exist in the current database and schema. If it does not, the statement returns SQLSTATE 42P01. That code means the view does not exist.
USERS
Lists all database users. Returns one column, username.
ROLES
Lists all roles. Returns one column, role_name.
GRANTS [ FOR user_name ]
Lists privilege grants. With no FOR clause, all grants are returned. With FOR user_name, the result is filtered to that user. The name may be unquoted or single-quoted. Returns user, scope, object, privileges, and grant_option. The scope value is TABLE, SCHEMA, or DATABASE. The grant_option value is YES or NO.
JOBS
Lists registered scheduled cron jobs. Returns name, schedule, sql, enabled, and last_run_unix. The schedule column is the cron expression. The sql column is the body it runs. The enabled column is a boolean. The last_run_unix column is the Unix timestamp of the last fire. A value of 0 means the job has never run.
SUBSCRIPTIONS
Lists durable subscriptions. Returns name, channel, last_ack_seq, next_seq, and ttl_days.
MEMORY STORES
Lists memory stores created with CREATE MEMORY STORE. Returns name, backing_table_id, embedding_dim, and distance. The distance value is cosine, l2, inner, or l1. The underscore spelling MEMORY_STORES is also accepted.
CHECKPOINT STORES
Lists checkpoint stores. Returns name, backing_table_id, and retention. The underscore spelling CHECKPOINT_STORES is also accepted.
ENTITY STORES
Lists entity stores. Returns name, backing_table_id, and ttl_days. The underscore spelling ENTITY_STORES is also accepted.
GRAPH STORES
Lists graph stores. Returns name, nodes_table_id, and edges_table_id. The underscore spelling GRAPH_STORES is also accepted.
DOCUMENT STORES
Lists document stores. Returns name, backing_table_id, embedding_dim, and distance_kind. The underscore spelling DOCUMENT_STORES is also accepted.
MESSAGE LOGS
Lists message logs. Returns name, backing_table_id, and ttl_days. The underscore spelling MESSAGE_LOGS is also accepted.
EVO_SESSIONS
Lists the sessions that are connected right now. Returns pid, username, database, state, and duration_ms. The pid column is the session id. The state value is active or idle. The duration_ms column is how many milliseconds the current query has been running. It is 0 when the session is idle.
EVO_PROCESSES
Works like EVO_SESSIONS. It adds a query column with the text of the session's current statement. The columns are pid, username, database, state, duration_ms, and query.
REPLICATION STATUS
Shows the streaming replication slots, one row per replica. Returns role, replica_id, my_lsn, confirmed_lsn, lag_bytes, active, and last_seen_us. The role value is primary, replica, or witness. When there are no replicas, a single summary row for the local role is returned. This form matches only when the statement text does not contain cluster.
REPLICATION LAG
Shows a short lag view, one row per slot. Returns replica_id and lag_bytes.
NODES
Lists nodes in a distributed cluster. Returns node_id, host, pg_port, and role. The role value is SELF or PEER. On a non-distributed server, it returns a single STANDALONE row for localhost:5433. The distributed-command path handles this form.
TABLE PLACEMENT
Lists every non-temporary table with the node that owns it. Returns table_name, schema, owner_node_id, and location. The location value is LOCAL or REMOTE. The distributed-command path handles this form.
SHARDS table_name
Lists the shards of a sharded table. Returns shard_name, ordinal, owner_node_id, range_bound, shard_type, and shard_key. The range_bound value is MAXVALUE for the open upper bound. The shard_type value is HASH or RANGE. It returns no rows if the table is not sharded. The distributed-command path handles this form.
configuration_parameter
Any SHOW form not matched above returns a single setting column. The known parameters are listed here. system_time_retention is the number of days of system-versioned history that is kept. A value of 0 means pruning is off. It is returned as system_time_retention_days. Other known parameters are max_connections, active_connections, statement_timeout / evo_statement_timeout, evo_lock_timeout, evo_idle_timeout, transaction_isolation, and search_path. server_version reports 15.0 (EvoSQL). max_identifier_length reports 63. standard_conforming_strings reports on. cluster reports the Raft status, which includes the role, term, leader, and LSN. buffer_pool_stats is a multi-column view of the hit rate and I/O. buffer_pool_size is also accepted as shared_buffers, innodb_buffer_pool_size, or db_cache_size. An unknown parameter does not raise an error. It returns one row with an empty setting value.

Examples

List databases. Then list the schemas and tables in the current database.

SHOW DATABASES;
SHOW SCHEMAS;
SHOW TABLES;

Rebuild the definition of a view.

SHOW CREATE VIEW active_users;

Show all grants. Then show only the grants held by one user.

SHOW GRANTS;
SHOW GRANTS FOR 'analyst';

Look at the connected sessions and the queries they are running.

SHOW EVO_PROCESSES;

List the shards of a sharded table on a distributed cluster.

SHOW SHARDS orders;

Read run-time parameters.

SHOW server_version;        -- 15.0 (EvoSQL)
SHOW max_connections;
SHOW buffer_pool_stats;     -- pool_pages, hits, misses, hit_rate, ...
SHOW system_time_retention; -- system_time_retention_days

Compatibility

The SQL standard has no SHOW statement. The run-time-parameter form of EvolutionDB is SHOW configuration_parameter. It follows the PostgreSQL convention. It supports several PostgreSQL-compatible names such as server_version, max_connections, standard_conforming_strings, search_path, and shared_buffers. To help other tools connect, SHOW server_version reports a PostgreSQL-style string. That string is 15.0 (EvoSQL).

The object-listing and store-listing forms are EvolutionDB extensions. These forms include SHOW DATABASES, SHOW TABLES, SHOW SCHEMAS, SHOW GRANTS, and the various ... STORES forms. Their shape is closer to MySQL-style SHOW listings than to PostgreSQL. The exact keywords and result columns are specific to EvolutionDB. PostgreSQL's SHOW ALL is not implemented. The SHOW SESSION and SHOW LOCAL qualifiers are not implemented either. Matching is substring-based, so these qualifiers are simply ignored. EvolutionDB does not raise an error for an unknown parameter. It returns an empty setting value instead. PostgreSQL raises an error in this case.
